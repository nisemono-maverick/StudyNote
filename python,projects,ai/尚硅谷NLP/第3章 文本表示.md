# 文本表示

> 核心问题：如何把非结构化的文本，转化为机器能处理的结构化向量？

---

## 3.1 概述
- 文本表示是将自然语言转化为计算机能够理解的数值形式
- 早期文本通常将整段文本编码为一个向量（如词袋模型），此类方法实现简单、计算高效，但是无法表达语序和小下文语义
- 文本第一步通常是分词和词表构建
	- **分词（Tokenization）** 是将原始文本切分为若干具有独立语义的最小单元（即token）的过程，是所有 NLP 任务的起点。
	- **词表（Vocabulary）** 是由语料库构建出的、包含模型可识别 token 的集合。词表中每个 token 都分配有唯一的 ID，并支持 token 与 ID 之间的双向映射。
![[Pasted image 20260604002113.png]]
- 后续模型构建过程中，模型会先对输入文本进行分词，再通过词表将其转换为词向量，最终生成结果。常常结果通过概率表示，系统选择最大出现概率，最后输出文本
![[Pasted image 20260604002130.png]]

## 3.2 分词
### 英文分词
#### 词级分词
- 按词语进行切分。空格、标点常常是天然的分割符
- OOV（Out-of-Vocabulary）：在模型推理阶段，输入文本中可能会出现不在预料库词表中的词
- 通常处理方式是将其统一替换为特殊标记，从而导致语义丢失
#### 字符级分词
- 以单个字符为最小单位进行分词
- 但是切成单个字符语义信息就会很弱了。输入序列也会很长
#### 子词级分词
- 将词语切成子词，如词根、前缀、后缀或常见词片段
- tiktokenizer.vercel.app 体验大模型分词算法
![[Pasted image 20260604225218.png]]
- 常见子词分词算法包括 BPE、WordPiece、Unigram Language Model
- BPE 的基本思想：
	- 在训练阶段，将语料拆成单个字符，构架初始词表；然后迭代的将出现频率高的相邻字符合并为子词单元加入词表，直到达到预设上限
	- 在分词阶段，先将语料拆成最小单位，然后根据训练所得规则逐步合并，直到无法继续
	- 详细介绍见[[BPE tokenization 算法]]

### 中文分词
#### 字符级分词
- 将文本按照单个汉字进行切分，每个汉字都视为独立的 token
- 由于汉字本身具有独立语义，所以字符级分词天然具有可行性
#### 词级分词
- 将中文文本按照完整词语进行切分，由于中文没有空格，词级分词依赖辞典、规则或模型来识别词语边界，与英文词级分词存在同样的问题
#### 子词级分词
- 同样可以套用类似 BPE 算法的思路对中文进行分词

- 在大模型出现之前，中文分词常常用分词器，进行词级分词

### 分词工具
- 基于辞典或模型的传统词级分词方法（jieba, HanLP）
- 基于子词建模算法的子词级分词方法 (HuggingFace Tokenizer、 SentencePiece、 tiktoken)

#### jieba 分词器
##### 分词模式
- 精确模式。试图将句子最精确地切开，适合文本分析。
- 只需导入 `jieba` 包，然后调用 `cut` 和 `lcut` 方法
```python
import jieba
text = "小明毕业于北京大学计算机系"
word_generator = jieba.cut(text) # cut方法返回一个生成器
for word in word_generator:
    print(word)
    
word_list = jieba.lcut(text) # lcut方法返回一个列表
print(word_list)
# ['小明', '毕业', '于', '北京大学', '计算机系']
```
- 全模式。把句子中所有的可以成词的词语都扫描出来
- 只需在 `cut` 和 `lcut` 方法中额外添加参数 `cut_all=True`
```python
word_list = jieba.lcut(text, cut_all=True)
print(word_list)
# ['小', '明', '毕业', '于', '北京', '北京大学', '大学', '计算', '计算机', '计算机系', '算机', '系']
```
- 搜索引擎模式。在精确模式基础上，对长词进一步切分，适合用于搜索引擎分词
- 调用 `cut_for_search` 方法
```python
word_list = jieba.lcut_for_search(text)
print(word_list)
# ['小明', '毕业', '于', '北京', '大学', '北京大学', '计算', '算机', '计算机', '计算机系']
```

## 3.3 词表示
### One-hot 编码
- 它将词汇表中的每个词映射为一个稀疏向量，向量的长度等于整个词表的大小。该词在对应的位置为 1，其他位置为 0。
- 无法体现词与词之间的语义关系
### 语义化词向量
#### Word2Vec 概述
- Word2Vec 的设计理念源自分布假设——即一个词的含义由它周围的词决定
- 基于这个假设，构建了一个简洁的神经网络模型，通过学习词语上下文之间的关系，学习每个词的向量表示
- 典型的 2 种模型结构：
	- CBOW：输入上下文去榆次中间词
	- Skip-gram：输入中心词，预测上下文的词
#### Word2Vec 原理
- 数据集：首先需要对原始文本进行分词，将连续文本转换为 token 序列，并将词语转换为 one-hot 编码，以便作为模型的输入和输出进行计算

- 前向传播过程如下：
	- 输入上下文词（乘坐、上班）：输入用 one-hot 向量表示
	- 查找词向量（$W_{in}$）：每个向量与参数矩阵 $W_{in}$ 相乘，取出“地铁”对应的词向量
	- 平均上下文向量：将多个上下文词向量取平均，得到一个整体的上下文表示
	- 预测中心词（$W_{out}$）: 平均后的上下文向量与参数矩阵 $W_{out}$ 相乘，得到整个词表的预测得分
	- softmax 输出：得分通过 softmax 转为概率分布

- 推理阶段，将 $W_{in}$ 与新的词语的 one-hot 向量相乘得到该词语的词向量 

#### 获取 Word2Vec 词向量
- 获取词向量的方式通常有 2 种
	- 直接使用他人公开发布的词向量
	- 在特定预料上自行训练

##### 使用公开词向量
- 使用`eyedVectors.load_word2vec_format`方法加载词向量文件
```python
from gensim.models import KeyedVectors
model_path = './data/sgns.weibo.word.bz2'
model = KeyedVectors.load_word2vec_format(model_path, binary=False)
```
- 使用加载模型的方法查看词向量维度、某词的词向量、两个向量的相似度
	- 其中，相似度计算是使用的余弦相似度 $\cos(\theta) = \frac{A \cdot B}{\|A\| \|B\|}$
	- 返回值介于 -1 到 1 之间，接近 1 说明语义相近，接近 0 说明无明显相关，接近 -1 说明语义相反
```python
print(model.vector_size) # 300
print(model['地铁']) # 词向量 一个300维的列表
similarity = model.similarity('地铁', '公交') # 计算两个词的相似度
print(similarity) # 0.65458214
```
- 同时可以使用 `most_similar` 方法找出与某个词醉相思的词
```python
similar_words = model.most_similar(positive=["上班"], topn=5)
print(similar_words) # [('赶点', 0.6254161596298218), ('下班', 0.6208967566490173), ('车限行', 0.5995197296142578), ('车限号', 0.5659466981887817), ('赶得', 0.5646229982376099)]
result = model.most_similar(positive=["爸爸", "女性"], negative=["男性"], topn=3)
print(result) # [('妈妈', 0.45269083976745605), ('星爸', 0.439832478761673), ('爸妈', 0.43170687556266785)]
```
##### 自行训练词向量
- 准备语料：训练需要已分词的文本序列
- 训练模型：gensim 提供十分方便的 API—— `Word2Vec` 直接进行训练
```python
sentences = [['我', '每天','乘坐', '地铁', '上班'], ['我','每天', '乘坐', '公交', '上班']]
from gensim.models import Word2Vec
model = Word2Vec(sentences,        # 已分词的句子序列
                 vector_size=100,  # 词向量维度 
                 window=5,         # 上下文窗口大小 
                 min_count=2,      # 最小词频，低于该频率的词会被忽略
                 sg=1,             # 训练算法，1表示使用skip-gram，0表示使用CBOW 
                 workers=4         # 训练线程数
                 )
```
- 保存词向量：`model.wv.save_word2vec_format('my_vectors.kv')`
- 然后再用 `eyedVectors.load_word2vec_format` 方法加载词向量文件就可以了

#### 应用 Word2Vec 词向量
- 在现代深度学习的 NLP 应用模型中，大多数任务的输入第一层都是嵌入层
- 嵌入层的参数矩阵有两种典型初始化方式
	- 随机初始化：嵌入向量是随机生成的，模型通过后续训练学习每个词的表示
	- 使用预训练词向量初始化：加载训练好的词向量模型
- 以下示例，如何使用预训练模型初始化 embedding 层
```python
import torch
import torch.nn as nn
from gensim.models import KeyedVectors
# 1. 加载预训练词向量
word_vectors = KeyedVectors.load_word2vec_format('./data/my_vectors.kv')
# 2. 构建词表和词向量矩阵
word2index = word_vectors.key_to_index   # 词到索引的映射
embedding_dim = word_vectors.vector_size # 词向量维度
num_embeddings = len(word2index)         # 词表大小
embedding_matrix = torch.zeros((num_embeddings, embedding_dim)) # 初始化词向量矩阵
for word, idx in word2index.items():
    embedding_matrix[idx] = torch.tensor(word_vectors[word]) # 将预训练词向量填入矩阵
# 3. 构建PyTorch的Embedding层
embedding_layer = nn.Embedding.from_pretrained(
    embedding_matrix,  # 预训练词向量矩阵
    freeze=False        # 是否冻结词向量，True表示不更新词向量
)
# 4. 示例：将词索引转换为向量
input_words = ["我", "喜欢", "乘坐", "地铁"] # 分词后的句子
input_indices = [word2index[word] for word in input_words] # 将词转换为索引
input_tensor = torch.tensor(input_indices) # 转换为PyTorch张量
# 5. 查询嵌入
output = embedding_layer(input_tensor) # 获取词向量
print(output) # 输出词向量张量
print(output.shape) # 输出词向量的形状 torch.Size([4, 100])
```
- 模型同样存在 OOV 问题，需要单独处理
```python
# 处理OOV问题
unk_token = '<UNK>'
index2word = [unk_token] + word_vectors.index_to_key
word2index = {word: idx for idx, word in enumerate(index2word)}
embedding_dim = word_vectors.vector_size # 词向量维度
num_embeddings = len(word2index)         # 词表大小
embedding_matrix = torch.zeros((num_embeddings, embedding_dim)) # 初始化词向量矩阵
for word, idx in word2index.items():
    if word in word_vectors:
        embedding_matrix[idx] = torch.tensor(word_vectors[word]) # 将预训练词向量填入矩阵
    else:
        embedding_matrix[idx] = torch.randn(embedding_dim) * 0.01 # 对于OOV词，使用随机初始化的向量
# 3. 构建PyTorch的Embedding层
embedding_layer = nn.Embedding.from_pretrained(
    embedding_matrix,  # 预训练词向量矩阵
    freeze=False        # 是否冻结词向量，True表示不更新词向量
)
# 4. 示例：将词索引转换为向量
input_words = ["我", "喜欢", "乘坐", "宇宙飞船"] # 分词后的句子
input_indices = [word2index.get(word, 0) for word in input_words] # 将词转换为索引
input_tensor = torch.tensor(input_indices) # 转换为PyTorch张量
# 5. 查询嵌入
output = embedding_layer(input_tensor) # 获取词向量
print(output) # 输出词向量张量
print(output.shape) # 输出词向量的形状 torch.Size([4, 100])
```
- 为了处理 OOV 问题，共计有 3 个地方改动
	- `index2word将unk_token加入`
	- 初始化 embedding 层权重时，加入判断，如果是 unk 则随机初始化
	- 将词转化为词向量时要用 get 方法去 `word2index` 中取，没取到就取 0 位置（unk_token）

### 上下文相关词表示
- Word2Vec 模型会给每个词分配一个固定的向量表示，无论他在句中的语境如何。这种表示成为静态词向量
- 然而，这种方式无法分辨“我爱吃苹果”“苹果发布了新产品”中的两个“苹果”的差别
- 上下文相关词表示（Contextual Word Representations），是指词语的向量表示会根据它所在的句子上下文动态变化，从而更好地捕捉其语义。这也为后续的 LSTM、BERT 等做出了预告
