# 词中的数学（Math with Words）

> 核心问题：如何用数学方法度量词语的重要性，并让计算机捕捉词与词之间的语义关系？

---

## 本章地图

本章的核心逻辑是**层层递进**的：

1. **词袋 + TF** → 把文档变成可计算的向量
2. **余弦相似度** → 用向量比较文档之间的相似性
3. **齐普夫定律** → 发现高频词是"噪声"，对区分文档没帮助
4. **TF-IDF** → 给"罕见但有区分力"的词更高权重，真正找到关键词

---

## 3.1 从词袋到 TF（词项频率）

### 词袋模型回顾
第2章已经学过：把文档拆成词，统计每个词出现了几次，得到一个**词频向量**。但这里有个坑：

> **长文档天然词频更高。** 一篇1000字的文章提到"Python" 10次，一条100字的 tweet 提到"Python" 2次，前者词频高只是因为文章长，不代表更重要。

### TF 的引入：消除文档长度偏差

**TF（Term Frequency）** 的核心思想：不要看原始次数，要看**比例**。

$$
TF(t, d) = \frac{\text{词 } t \text{ 在文档 } d \text{ 中出现的次数}}{\text{文档 } d \text{ 的总词数}}
$$

这样每个文档的向量分量之和为 1，长文档和短文档才能真正可比。

**⚠️ 注意**：书里第3章早期的示例代码写的是 `value / len(lexicon)`（除以词汇表大小），这是**不标准的**。正确的归一化应该除以**当前文档长度** `len(tokens)`。下面 3.2 的代码已修正。

## 3.2 向量化实战：从文本到 TF 向量

### 代码拆解

```python
from collections import Counter, OrderedDict
from nltk.tokenize import TreebankWordTokenizer
import copy

tokenizer = TreebankWordTokenizer()

# 3 个示例文档
docs = [
    "The faster Harry got to the store, the faster and faster Harry would get home.",
    "Harry is hairy and faster than Jill.",
    "Jill is not as hairy as Harry."
]

# ---------- 步骤 1：构建词汇表（lexicon）----------
doc_tokens = []
for doc in docs:
    doc_tokens += [sorted(tokenizer.tokenize(doc.lower()))]

all_doc_tokens = sum(doc_tokens, [])          # 所有文档的词拼在一起
lexicon = sorted(set(all_doc_tokens))          # 去重排序，得到词汇表
# len(lexicon) == 18
# [',', '.', 'and', 'as', 'faster', 'get', 'got', 'hairy', 'harry', 'home', 
#  'is', 'jill', 'not', 'store', 'than', 'the', 'to', 'would']

# ---------- 步骤 2：把每篇文档转成 TF 向量 ----------
zero_vector = OrderedDict((token, 0) for token in lexicon)

doc_vectors = []
for doc in docs:
    vec = copy.copy(zero_vector)
    tokens = tokenizer.tokenize(doc.lower())    # 当前文档的所有词
    token_counts = Counter(tokens)              # 每个词出现了几次
    
    # ✅ 正确的 TF 归一化：除以当前文档长度，而不是词汇表大小
    doc_len = len(tokens)
    for key, value in token_counts.items():
        vec[key] = value / doc_len
    
    doc_vectors.append(vec)

# 文档1 的非零项（仅示意）
# {
#   ',': 0.059, '.': 0.059, 'and': 0.059, 
#   'faster': 0.176, 'get': 0.059, 'got': 0.059,
#   'harry': 0.118, 'home': 0.059, 'store': 0.059,
#   'the': 0.176, 'to': 0.059, 'would': 0.059
# }
```

### 这 3 个向量到底是什么？

每个向量是 **18 维的数值指纹**，代表一篇文档：

| 维度 | 含义 | 数值意义 |
|------|------|---------|
| 第 1 维 | `','` | 逗号占该文档的比例 |
| 第 5 维 | `'faster'` | "faster" 占该文档的比例 |
| 第 9 维 | `'harry'` | "harry" 占该文档的比例 |

**关键洞察**：
- 文档1（Harry faster...）中 `faster` 出现 3 次，文档长度 17 → TF = 3/17 ≈ **0.176**
- 文档2（Harry is hairy...）中 `faster` 只出现 1 次，文档长度 7 → TF = 1/7 ≈ **0.143**

这样一归一化，两篇不同长度的文档就可以公平比较了。

## 3.3 余弦相似度（Cosine Similarity）

### 为什么需要它？

有了向量，自然想问：**文档 A 和文档 B 有多像？**

直接想法是用**点积**，但点积受向量长度影响（长文档天然高点积）。**余弦相似度**通过归一化消除了长度影响，只比较**方向**。

### 公式

$$
\text{cosine-sim}(A, B) = \frac{A \cdot B}{\|A\| \times \|B\|} = \frac{\sum_{i=1}^{n} A_i B_i}{\sqrt{\sum_{i=1}^{n} A_i^2} \times \sqrt{\sum_{i=1}^{n} B_i^2}}
$$

### 代码实现

```python
import numpy as np

def cosine_similarity(a, b):
    """计算两个向量的余弦相似度"""
    dot = np.dot(a, b)
    norm_a = np.linalg.norm(a)
    norm_b = np.linalg.norm(b)
    if norm_a == 0 or norm_b == 0:
        return 0.0
    return dot / (norm_a * norm_b)

# 示例：假设 doc1 和 doc2 的 TF 向量（仅取数值部分，按词汇表顺序）
v1 = np.array([0.059, 0.059, 0.059, 0, 0.176, 0.059, 0.059, 0, 0.118, 
               0.059, 0, 0, 0, 0.059, 0, 0.176, 0.059, 0.059])
v2 = np.array([0, 0, 0.143, 0, 0.143, 0, 0, 0.143, 0.143, 
               0, 0.143, 0.143, 0, 0, 0.143, 0, 0, 0])

sim = cosine_similarity(v1, v2)
# 结果 ≈ 0.42，说明两篇文档有一定相似性（都提到了 harry 和 faster）
```

### 结果解读

| 余弦值 | 含义 |
|--------|------|
| **1** | 方向完全相同，两篇文档"极度相似" |
| **0** | 完全正交，两篇文档没有共同词 |
| **-1** | 方向完全相反（在 TF 非负的情况下不可能出现） |

### 直观理解

- 文档1 和 文档2 都提到了 `harry` 和 `faster` → 夹角小 → **余弦相似度高**
- 文档1 和 文档3 几乎没有共同词 → 夹角接近 90° → **余弦相似度接近 0**

## 3.4 齐普夫定律（Zipf's Law）

### 定律内容

> 在一个自然语言语料库中，**任意一个词的出现频率与它在频率表中的排名成反比**。

简单说：第 1 高频的词出现次数 ≈ 第 2 高频词的两倍 ≈ 第 3 高频词的三倍……

$$
f(r) \propto \frac{1}{r}
$$

其中 $r$ 是排名，$f(r)$ 是第 $r$ 名词的频率。

### 对 NLP 的启示

| 排名 | 典型词 | 问题 |
|------|--------|------|
| Top 1-10 | `the`, `is`, `and`, `a`... | 几乎每篇文档都有，**对区分文档毫无帮助** |
| 中间段 | `faster`, `hairy` | 有一定区分力 |
| 长尾 | `aerodynamics`, `construction` | 只在特定文档出现，**区分力最强** |

**核心结论**：
- 高频词 ≠ 重要词
- 真正能帮助区分不同文档的，是那些**不常见但具有领域特征**的词

### 为什么需要"对数缩放"？

齐普夫定律是一个**幂律分布**（少数词占用了绝大多数频次），直接用原始频次会导致高频词压倒一切。取对数可以**压缩尺度**，让低频词也有发言权。

## 3.5 TF-IDF：关键词的终极度量

### 从直觉到公式

TF 已经解决了"文档长度偏差"，但还没解决"高频噪声"问题。

**IDF（Inverse Document Frequency，逆文档频率）** 的直觉：
- 如果某个词**只在少数文档中出现**，它很可能是这些文档的"关键词"
- 如果某个词**在所有文档中都出现**，它没什么区分价值

$$
IDF(t, D) = \log \frac{N}{|\{d \in D : t \in d\}|}
$$

其中：
- $N$ = 文档总数
- $|\{d \in D : t \in d\}|$ = 包含词 $t$ 的文档数

**⚠️ 平滑处理**：实际工程中分母会 +1（拉普拉斯平滑），防止某个词在所有文档都没出现时分母为 0：

$$
IDF(t, D) = \log \frac{N}{1 + |\{d \in D : t \in d\}|}
$$

### TF-IDF 公式

$$
TF\text{-}IDF(t, d, D) = TF(t, d) \times IDF(t, D)
$$

### 为什么乘起来？

| 情况 | TF | IDF | TF-IDF | 解读 |
|------|----|-----|--------|------|
| 文档内高频，且只在少数文档出现 | 高 | 高 | **很高** | ✅ 真正重要的关键词 |
| 文档内高频，但在所有文档都出现 | 高 | 低 | 低 | ❌ 停用词/噪声 |
| 文档内低频，且只在少数文档出现 | 低 | 高 | 中等 | 有一定区分力 |
| 文档内低频，且在很多文档出现 | 低 | 低 | 很低 | 无关紧要 |

### sklearn 实战

```python
from sklearn.feature_extraction.text import TfidfVectorizer

docs = [
    "The faster Harry got to the store, the faster and faster Harry would get home.",
    "Harry is hairy and faster than Jill.",
    "Jill is not as hairy as Harry."
]

# 构建 TF-IDF 矩阵
vectorizer = TfidfVectorizer(stop_words='english')  # 自动去掉停用词
tfidf_matrix = vectorizer.fit_transform(docs)

# 查看词汇表
print(vectorizer.get_feature_names_out())

# 查看 TF-IDF 矩阵（稀疏矩阵）
print(tfidf_matrix.toarray())
```

### 相关度排序

有了 TF-IDF 向量，就可以用余弦相似度来做**文档检索**：

```python
from sklearn.metrics.pairwise import cosine_similarity

# 计算文档之间的两两相似度
sim_matrix = cosine_similarity(tfidf_matrix)
# sim_matrix[i][j] = 文档 i 和文档 j 的余弦相似度
```

**工程 tip**：搜索引擎一般使用**倒排索引**（Inverted Index），可以在 $O(1)$ 时间内定位包含关键词的文档，而不是遍历所有文档。

## 3.6 其它 TF 加权方案对比

除了标准的 TF-IDF，工业界和信息检索领域还有几种常见的变体：

| 方案 | 公式 | 核心思想 | 适用场景 |
|------|------|---------|---------|
| **原始计数** | $w_{ij} = f_{ij}$ | 不做任何加权 | 简单任务、短文本 |
| **归一化 TF** | $w_{ij} = \frac{f_{ij}}{\sum_k f_{kj}}$ | 消除文档长度影响 | 文本分类、聚类 |
| **对数 TF** | $w_{ij} = 1 + \log(f_{ij})$ | 压缩高频词的压倒性优势 | 长文档分析 |
| **TF-IDF（标准）** | $w_{ij} = \text{TF}_{ij} \times \log\frac{N}{n_j}$ | 同时考虑文档内频率和文档间稀缺度 | 关键词提取、信息检索 |
| **TF-IDF（对数版）** | $w_{ij} = \log(1 + f_{ij}) \times \log\frac{N}{n_j}$ | TF 也用对数压缩 | 齐普夫分布明显的语料 |
| **TF-ICF** | $w_{ij} = \text{TF}_{ij} \times \log\frac{C}{c_j}$ | 用**类别频率**代替文档频率 | 文本分类（$C$=类别总数，$c_j$=包含词 $j$ 的类别数）|
| **Okapi BM25** | 见下方 | 考虑文档长度饱和效应，非线性 TF | 搜索引擎（Elasticsearch、Lucene）|

### Okapi BM25 详解

BM25 是 TF-IDF 的改进版，广泛应用于现代搜索引擎。它解决了 TF-IDF 的两个问题：
1. **TF 无限增长**：TF-IDF 中线性增长不合理，一个词出现 100 次不应是 10 次的 10 倍重要
2. **文档长度偏差**：长文档的 IDF 惩罚不够精确

**BM25 公式**：

$$
score(D, Q) = \sum_{i=1}^{n} IDF(q_i) \cdot \frac{f(q_i, D) \cdot (k_1 + 1)}{f(q_i, D) + k_1 \cdot \left(1 - b + b \cdot \frac{|D|}{avgdl}\right)}
$$

其中：
- $f(q_i, D)$：词 $q_i$ 在文档 $D$ 中的出现次数
- $|D|$：文档 $D$ 的长度（词数）
- $avgdl$：语料库中文档的平均长度
- $k_1$：控制 TF 饱和度的参数（通常取 **1.2**）
  - $k_1$ 越大，TF 的影响越接近线性
  - $k_1$ 越小，TF 越快饱和
- $b$：控制文档长度归一化的参数（通常取 **0.75**）
  - $b=1$：完全按文档长度归一化
  - $b=0$：不考虑文档长度

**BM25 vs TF-IDF 的关键区别**：

| 特性 | TF-IDF | BM25 |
|------|--------|------|
| TF 增长 | 线性 | 非线性（快速饱和）|
| 文档长度处理 | 简单归一化 | 精细的参数化归一化 |
| 参数数量 | 无 | $k_1$, $b$ 可调 |
| 典型应用 | 传统 NLP 流水线 | Elasticsearch、Solr 等搜索引擎 |

## 关键概念速查

| 术语 | 英文 | 含义 |
|------|------|------|
| **词袋模型** | Bag of Words (BoW) | 忽略词序，只统计词频的文本表示方法 |
| **词项频率** | TF (Term Frequency) | 词在文档中出现次数 / 文档总词数 |
| **逆文档频率** | IDF (Inverse Document Frequency) | 衡量词对区分文档的稀缺程度 |
| **TF-IDF** | TF-IDF | TF × IDF，综合衡量词在文档中的重要性 |
| **余弦相似度** | Cosine Similarity | 两个向量夹角的余弦值，用于度量文档相似度 |
| **齐普夫定律** | Zipf's Law | 词频与排名成反比的统计规律 |
| **词汇表** | Lexicon / Vocabulary | 语料库中所有不重复词的集合 |
| **停用词** | Stop Words | 高频但信息量极低的词（如 the, is, and） |
| **倒排索引** | Inverted Index | 词 → 文档列表的映射，用于快速检索 |
| **拉普拉斯平滑** | Laplace Smoothing | 分母 +1 防止除零 |
| **BM25** | Okapi BM25 | 信息检索中 TF-IDF 的改进版，引入非线性 TF 饱和 |
