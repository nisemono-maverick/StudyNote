# 词频背后的语义（Finding Meaning in Word Counts）

> 核心问题：第3章我们把文档变成了 TF-IDF 向量，但这些向量仍然是**高维且稀疏**的。如何从海量词频中挖掘出**隐藏的语义关系**？

---

## 本章地图

本章的核心逻辑：**把高维稀疏的"词-文档"矩阵，通过数学分解，压缩到低维稠密的"语义空间"**。

1. **词-文档矩阵** → 把语料库整理成一个大矩阵
2. **SVD（奇异值分解）** → 矩阵分解的数学工具
3. **LSA（潜在语义分析）** → 用 SVD 发现"潜在主题"
4. **PCA** → 另一种降维视角
5. **语义相似度** → 在降维后的空间中重新理解"词的距离"

---

## 4.1 词-文档矩阵（Term-Document Matrix）

### 回顾：我们已有的数据

第3章结束后，语料库可以被表示为一个矩阵：

| | 文档1 | 文档2 | 文档3 | ... | 文档N |
|---|:---:|:---:|:---:|:---:|:---:|
| **the** | 0.176 | 0 | 0 | ... | 0.05 |
| **faster** | 0.176 | 0.143 | 0 | ... | 0 |
| **harry** | 0.118 | 0.143 | 0.2 | ... | 0 |
| **hairy** | 0 | 0.143 | 0.2 | ... | 0 |
| ... | ... | ... | ... | ... | ... |
| **aerodynamics** | 0 | 0 | 0 | ... | 0.3 |

- **行** = 词汇表中的词（可能几万到几百万维）
- **列** = 文档
- **值** = TF 或 TF-IDF

### 问题：这个矩阵太"笨"了

1. **维度灾难**：词汇表动辄几万维，矩阵极其稀疏
2. **同义词无法关联**：`car` 和 `automobile` 在不同行，计算机不知道它们语义相近
3. **多义词无法区分**：`bank`（银行）和 `bank`（河岸）在同一行，计算机无法区分不同含义

> **LSA 的核心洞察**：词与词之间的关系，文档与文档之间的关系，其实可以由少数几个"潜在主题"（Latent Topics）来解释。我们不需要几万维的词汇空间，也许只需要 100~300 维的"语义空间"。

---

## 4.2 奇异值分解（SVD）

### 什么是 SVD？

**SVD（Singular Value Decomposition）** 是一种矩阵分解技术。在 LSA 中，我们实际使用的是**截断 SVD（Truncated SVD）**，把词-文档矩阵 $A$ 分解成三个矩阵的乘积：

$$
A_{m \times n} \approx U_{m \times k} \cdot \Sigma_{k \times k} \cdot V^T_{k \times n}
$$

其中：
- $A$：原始词-文档矩阵（$m$ 个词 × $n$ 篇文档）
- $U$：**词-主题矩阵**（$m$ 词 × $k$ 主题）。每一列是一个"主题"，每一行是词在该主题上的权重
- $\Sigma$：**奇异值对角矩阵**（$k$ × $k$）。对角线上的值表示每个主题的重要程度（从大到小排列）
- $V^T$：**主题-文档矩阵**（$k$ 主题 × $n$ 文档）。每一行是一个"主题"，每一列是文档在该主题上的权重

> 从纯数学角度看，完整 SVD 的分解形式是 $A = U_{m \times m} \, \Sigma_{m \times n} \, V^T_{n \times n}$，其中 $U$ 和 $V$ 都是正交方阵。但在 LSA 中，我们只保留前 $k$ 个最大的奇异值及对应的奇异向量，因此实际使用的是 $U_k$、$\Sigma_k$、$V_k^T$。

### 直观理解

截断 SVD 本质上是在做这件事：

```
原始矩阵 A          ≈    词-主题矩阵 U     ×     主题强度 Σ     ×     主题-文档矩阵 V^T
(m 词 × n 文档)        (m 词 × k 主题)          (k 主题 × k 主题)       (k 主题 × n 文档)
```

**关键洞察**：
- $U$ 的每一列是一个"潜在主题"，告诉我们哪些词经常一起出现
- $V^T$ 的每一行是文档在这个主题上的"得分"
- $\Sigma$ 告诉我们哪些主题更重要（值越大越重要）

### 代码实战：用 sklearn 做截断 SVD

```python
from sklearn.decomposition import TruncatedSVD
from sklearn.feature_extraction.text import TfidfVectorizer

# 示例文档
docs = [
    "The faster Harry got to the store, the faster and faster Harry would get home.",
    "Harry is hairy and faster than Jill.",
    "Jill is not as hairy as Harry."
]

# 先转成 TF-IDF 矩阵
vectorizer = TfidfVectorizer(stop_words='english')
tfidf_matrix = vectorizer.fit_transform(docs)  # shape: (3, 8)

# 截断 SVD：保留 2 个主题
svd = TruncatedSVD(n_components=2, random_state=42)
lsa_matrix = svd.fit_transform(tfidf_matrix)  # shape: (3, 2)

# 查看奇异值（主题强度）
print("奇异值:", svd.singular_values_)
# 奇异值: [0.52 0.34]

# 查看词-主题矩阵 U
terms = vectorizer.get_feature_names_out()
for i, comp in enumerate(svd.components_):
    terms_in_comp = zip(terms, comp)
    sorted_terms = sorted(terms_in_comp, key=lambda x: x[1], reverse=True)[:5]
    print(f"\n主题 {i}:")
    for term, weight in sorted_terms:
        print(f"  {term}: {weight:.3f}")
```

**输出解读**：
- `lsa_matrix` 把每篇文档从原来的 8 维压缩到了 2 维
- `singular_values_` 显示第 1 个主题比第 2 个主题更强
- `components_` 显示每个主题由哪些词构成

### ⚠️ 注意：SVD 与 LSA 的关系

| | SVD | LSA |
|---|-----|-----|
| **本质** | 纯数学的矩阵分解技术 | 将 SVD 应用于 NLP 的**思想/框架** |
| **关系** | 工具 | 应用该工具的领域方法 |
| **说法** | "对矩阵做 SVD" | "用 LSA 做语义分析" |

> 很多文献把两者混用，但严格来说：**LSA = 把 SVD 用在词-文档矩阵上，以发现潜在语义结构**。

## 4.3 潜在语义分析（LSA）

### 核心思想

LSA 假设：词与词之间的**共现关系**蕴含了语义信息。如果两个词经常出现在相同的文档中，它们很可能是语义相关的（即使字面上不同，如 `car` 和 `automobile`）。

**LSA 解决的核心问题**：
1. **同义词问题**：`car` 和 `automobile` 在原始矩阵中是不同的行，但在 LSA 的语义空间中可能指向同一个方向
2. **多义词问题**：`bank` 在原始矩阵中只有一行，但在 LSA 中可能被拆分到不同的主题维度上

### 降维后的语义空间

假设原始词汇表有 10,000 个词，我们有 1,000 篇文档：

| | 原始空间 | LSA 语义空间 |
|---|---------|-------------|
| **维度** | 10,000（词） | 100~300（主题） |
| **稀疏度** | 极度稀疏（每个文档只含少量词） | 稠密（每个文档在所有主题上都有得分） |
| **可解释性** | 每一维是一个具体词 | 每一维是一个"抽象主题" |
| **相似度计算** | 只有共同出现的词才有非零点积 | 即使没共同词，也可能在语义空间中相近 |

### 实战：用 LSA 计算文档相似度

```python
from sklearn.metrics.pairwise import cosine_similarity

# 计算降维后的余弦相似度
sim_matrix = cosine_similarity(lsa_matrix)
print(sim_matrix)

# [[1.   0.72 0.31]
#  [0.72 1.   0.45]
#  [0.31 0.45 1.  ]]
```

**结果解读**：
- 文档1 和 文档2 的相似度从原始 TF-IDF 的 ~0.42 提升到了 **0.72**
- 为什么？因为 LSA 发现了它们共享的潜在主题（`harry`/`faster`）
- 文档3 和 其他文档的相似度较低，因为它的主题偏向 `jill`/`hairy`

### LSA 的局限性

| 问题 | 说明 |
|------|------|
| **主题不可解释** | SVD 分解出的"主题"是数学上的正交基，不一定对应人类理解的主题 |
| **无法处理新词** | 新文档需要重新计算整个 SVD，不能增量更新 |
| **正负值问题** | SVD 结果中有负数，难以解释为"词频"或"概率" |
| **线性假设** | 假设语义关系是线性的，无法捕捉复杂的非线性模式 |

## 4.4 主成分分析（PCA）

### PCA 与 SVD 的关系

**PCA（Principal Component Analysis）** 是另一种降维技术，与 SVD 有密切联系：

| | PCA | SVD |
|---|-----|-----|
| **输入** | 通常对**特征-样本**矩阵操作 | 直接对原始矩阵操作 |
| **数学本质** | 对协方差矩阵做特征分解 | 直接对数据矩阵做分解 |
| **中心化处理** | **必须**先减去均值 | 可以不做 |
| **适用场景** | 稠密数据（如图像、传感器数据） | 稀疏数据（如文本的 TF-IDF 矩阵） |
| **在 LSA 中** | 不太常用（因为 TF-IDF 矩阵稀疏） | 标准选择 |

**核心联系**：对中心化后的数据矩阵做 SVD，得到的右奇异向量 $V$ 就是 PCA 的主成分。

### PCA 的直觉

PCA 寻找数据中方差最大的方向：
1. 第 1 主成分：数据散布最"开"的方向
2. 第 2 主成分：与第 1 主成分正交的方向中，方差最大的
3. 以此类推...

**在文本上的局限**：
- TF-IDF 矩阵极度稀疏，中心化后破坏稀疏性，计算代价高
- 文本数据的"方差"不如图像数据直观
- 因此 NLP 中更常用 SVD/LSA，而不是 PCA

### 代码：PCA 对比（了解即可）

```python
from sklearn.decomposition import PCA

# ⚠️ PCA 需要稠密矩阵，且必须先中心化
# 对 TF-IDF 矩阵不太友好，仅作演示
tfidf_dense = tfidf_matrix.toarray()

pca = PCA(n_components=2)
pca_matrix = pca.fit_transform(tfidf_dense)

print(" explained variance ratio:", pca.explained_variance_ratio_)
# 显示前 2 个主成分解释了总方差的多少比例
```

> 💡 **工程建议**：处理文本的 TF-IDF 矩阵时，优先用 `TruncatedSVD`（即 LSA），而不是 `PCA`。

## 4.5 主题建模与语义搜索（概述）

### 从 LSA 到更现代的主题模型

LSA 是主题建模的鼻祖，但后续发展出了更强大的方法：

| 模型 | 核心改进 | 特点 |
|------|---------|------|
| **LSA** | SVD 降维 | 线性、确定性的分解 |
| **pLSA** | 引入概率框架 | 把主题视为隐变量，用 EM 算法求解 |
| **LDA** | 加入 Dirichlet 先验 | 更完整的概率生成模型，主题更可解释 |
| **神经网络主题模型** | 用深度学习 | 如 BERTopic，结合预训练语言模型 |

> 第 4.5 节的详细内容（LDA 等）将在后续笔记中展开，参见 [[LDA|LDA 笔记]]。

### 语义搜索的基本流程

LSA 降维后的向量可以用于**语义搜索**（Semantic Search）：

```
用户查询 → 转成 TF-IDF 向量 → 用同样的 SVD 模型投影到语义空间 
→ 与文档向量计算余弦相似度 → 返回最相似的文档
```

**与传统关键词搜索的区别**：
- 关键词搜索：必须包含相同的词（`car` 搜不到 `automobile`）
- 语义搜索：在语义空间中相近即可（`car` 和 `automobile` 距离很近）

## 关键概念速查

### 4.1 ~ 4.2 矩阵分解基础

| 术语 | 英文 | 含义 |
|------|------|------|
| 词-文档矩阵 | Term-Document Matrix | 行是词、列是文档、值是词频的矩阵 |
| SVD | Singular Value Decomposition | 奇异值分解，将矩阵分解为 U·Σ·V^T |
| 截断 SVD | Truncated SVD | 只保留前 k 个奇异值的近似分解 |
| 奇异值 | Singular Value | Σ 对角线上的值，表示对应主题的重要性 |
| 左奇异向量 | Left Singular Vector | U 矩阵的列，代表"词-主题"关系 |
| 右奇异向量 | Right Singular Vector | V^T 矩阵的行，代表"主题-文档"关系 |

### 4.3 潜在语义分析

| 术语 | 英文 | 含义 |
|------|------|------|
| LSA | Latent Semantic Analysis | 潜在语义分析，用 SVD 发现词与文档的潜在主题 |
| 潜在主题 | Latent Topic | SVD 分解后得到的抽象语义维度 |
| 语义空间 | Semantic Space | 降维后的低维向量空间，距离代表语义相似度 |
| 同义词关联 | Synonymy | LSA 能发现字面不同但语义相近的词 |
| 多义词消解 | Polysemy | LSA 能将多义词分散到不同主题维度 |

### 4.4 主成分分析

| 术语 | 英文 | 含义 |
|------|------|------|
| PCA | Principal Component Analysis | 主成分分析，寻找方差最大的投影方向 |
| 主成分 | Principal Component | 数据方差最大的正交方向 |
|  explained variance | 解释方差 | 每个主成分保留了多少原始数据的信息 |
| 中心化 | Centering | 每个特征减去其均值，PCA 的必要步骤 |
