# 词频背后的语义（Finding Meaning in Word Counts）

> 核心问题：第3章我们把文档变成了 TF-IDF 向量，但这些向量仍然是**高维且稀疏**的。如何从海量词频中挖掘出**隐藏的语义关系**？

---

## 本章地图

本章的核心逻辑：**把高维稀疏的"词-文档"矩阵，通过数学分解，压缩到低维稠密的"语义空间"**。

1. **词-文档矩阵** → 把语料库整理成一个大矩阵
2. **SVD（奇异值分解）** → 矩阵分解的数学工具
3. **LSA（潜在语义分析）** → 用 SVD 发现"潜在主题"
4. **PCA** → 另一种降维视角
5. **语义相似度** → 在降维后的空间中重新理解"词的距离"

按原书目录：
- [ ] 4.1 从词频到语义空间
- [ ] 4.2 奇异值分解（SVD）
- [ ] 4.3 潜在语义分析（LSA）
- [ ] 4.4 主成分分析（PCA）
- [ ] 4.5 主题建模与语义搜索

## 4.1 词-文档矩阵（Term-Document Matrix）

### 回顾：我们已有的数据

第3章结束后，语料库可以被表示为一个矩阵：

| | 文档1 | 文档2 | 文档3 | ... | 文档N |
|---|:---:|:---:|:---:|:---:|:---:|
| **the** | 0.176 | 0 | 0 | ... | 0.05 |
| **faster** | 0.176 | 0.143 | 0 | ... | 0 |
| **harry** | 0.118 | 0.143 | 0.2 | ... | 0 |
| **hairy** | 0 | 0.143 | 0.2 | ... | 0 |
| ... | ... | ... | ... | ... | ... |
| **aerodynamics** | 0 | 0 | 0 | ... | 0.3 |

- **行** = 词汇表中的词（可能几万到几百万维）
- **列** = 文档
- **值** = TF 或 TF-IDF

### 问题：这个矩阵太"笨"了

1. **维度灾难**：词汇表动辄几万维，矩阵极其稀疏
2. **同义词无法关联**：`car` 和 `automobile` 在不同行，计算机不知道它们语义相近
3. **多义词无法区分**：`bank`（银行）和 `bank`（河岸）在同一行，计算机无法区分不同含义

> **LSA 的核心洞察**：词与词之间的关系，文档与文档之间的关系，其实可以由少数几个"潜在主题"（Latent Topics）来解释。我们不需要几万维的词汇空间，也许只需要 100~300 维的"语义空间"。

### 一个 [[LDA|LDA（线性判别分析）]] 分类器
- 模型训练只有 3 个步骤
	- 计算某个类中所有 TF-IDF 矩阵的质心
	- 计算不在该类中所有 TF-IDF 矩阵的质心
	- 计算两个质心之间的向量差
```python
import pandas as pd
from nlpia.data.loaders import get_data
pd.options.display.width = 120
sms = get_data('sms-spam')
index = ['sms{}{}'.format(i, '!'*j) for (i, j) in zip(range(len(sms)), sms.spam)]
sms = pd.DataFrame(sms.values, columns=sms.columns, index=index)
len(sms) #4837
sms.spam.sum() #638
sms.head(6)
#       spam                                               text
# sms0     0  Go until jurong point, crazy.. Available only ...
# sms1     0                      Ok lar... Joking wif u oni...
# sms2!    1  Free entry in 2 a wkly comp to win FA Cup fina...
# sms3     0  U dun say so early hor... U c already then say...
# sms4     0  Nah I don't think he goes to usf, he lives aro...
# sms5!    1  FreeMsg Hey there darling it's been 3 week's n...
```
- 垃圾短信息数据集，总数 4837 条，其中 638 条被标注为垃圾信息
- 下边对这些信息进行分词
```python
# 分词
from sklearn.feature_extraction.text import TfidfVectorizer
from nltk.tokenize.casual import casual_tokenize
tfidf_model = TfidfVectorizer(tokenizer=casual_tokenize)
tfidf_docs = tfidf_model.fit_transform(raw_documents=sms.text).toarray()
tfidf_docs.shape # (4837, 9232)
```

```python
# LDA训练
mask = sms.spam.astype(bool).values
spam_centroid = tfidf_docs[mask].mean(axis=0)
ham_centroid = tfidf_docs[~mask].mean(axis=0)
spam_centroid.round(2) # array([0.06, 0.  , 0.  , ..., 0.  , 0.  , 0.  ])
ham_centroid.round(2) # array([0.02, 0.01, 0.  , ..., 0.  , 0.  , 0.  ])
spamminess_score = tfidf_docs.dot(spam_centroid - ham_centroid)
spamminess_score.round(2) # array([-0.01, -0.02,  0.04, ..., -0.01, -0.  ,  0.  ])
```
- 这个原始的 spamminess_score 是非垃圾类质心到垃圾类质心的直线距离

### ⚠️ 这段代码的本质：一个「偷懒版」的 LDA

上面的实现**并不是标准 LDA**，而是一个**基于质心的线性分类器**。它借用了 LDA「找一个方向投影」的思想，但省略了 LDA 最核心的优化过程。

#### 每一步在做什么

| 代码 | 本质 |
|------|------|
| `spam_centroid = tfidf_docs[mask].mean(axis=0)` | 计算垃圾短信在 TF-IDF 空间中的「平均位置」 |
| `ham_centroid = tfidf_docs[~mask].mean(axis=0)` | 计算正常短信在 TF-IDF 空间中的「平均位置」 |
| `spam_centroid - ham_centroid` | 把两个质心连成一条向量，当作「垃圾方向」 |
| `tfidf_docs.dot(...)` | 把所有文档投影到这个方向上，看「有多像垃圾」 |

#### 为什么它「不是」标准 LDA

| | 这段代码 | 标准 LDA（`sklearn.discriminant_analysis.LinearDiscriminantAnalysis`） |
|---|---------|---------------------------------------------------------------|
| **训练过程** | 只算质心，完事儿 | 计算类内散度矩阵 $S_w$、类间散度矩阵 $S_b$，求解广义特征值问题 |
| **投影方向** | 直接用质心差向量 `spam - ham` | 求解 $S_w^{-1}S_b$ 的最大特征向量，保证「类间开、类内紧」 |
| **优化** | ❌ 没有最大化/最小化过程 | ✅ 显式最大化 $\frac{w^T S_b w}{w^T S_w w}$ |
| **假设** | 隐含假设：两类分布形状相同且是球形 | 假设：两类服从高斯分布，协方差矩阵相同（但不必是球形） |

#### 一句话总结

> 这段代码**没有学习最优投影方向**，而是**粗暴地把「两个质心的连线」当成了最好的分界线**。它省掉了 LDA 的数学优化，换来了极简的实现；代价是——如果两类数据分布杂乱、有重叠，这条线的分类效果会远不如真正的 LDA。

#### 如果想换成标准 LDA

```python
from sklearn.discriminant_analysis import LinearDiscriminantAnalysis

lda = LinearDiscriminantAnalysis()
lda.fit(tfidf_docs, sms.spam)
spamminess_score_lda = lda.decision_function(tfidf_docs)
```

## 4.2 奇异值分解（SVD）

### 什么是 SVD？

**SVD（Singular Value Decomposition）** 是一种矩阵分解技术。在 LSA 中，我们实际使用的是**截断 SVD（Truncated SVD）**，把词-文档矩阵 $A$ 分解成三个矩阵的乘积：

$$
A_{m \times n} \approx U_{m \times k} \cdot \Sigma_{k \times k} \cdot V^T_{k \times n}
$$

其中：
- $A$：原始词-文档矩阵（$m$ 个词 × $n$ 篇文档）
- $U$：**词-主题矩阵**（$m$ 词 × $k$ 主题）。每一列是一个"主题"，每一行是词在该主题上的权重
- $\Sigma$：**奇异值对角矩阵**（$k$ × $k$）。对角线上的值表示每个主题的重要程度（从大到小排列）
- $V^T$：**主题-文档矩阵**（$k$ 主题 × $n$ 文档）。每一行是一个"主题"，每一列是文档在该主题上的权重

> 从纯数学角度看，完整 SVD 的分解形式是 $A = U_{m \times m} \, \Sigma_{m \times n} \, V^T_{n \times n}$，其中 $U$ 和 $V$ 都是正交方阵。但在 LSA 中，我们只保留前 $k$ 个最大的奇异值及对应的奇异向量，因此实际使用的是 $U_k$、$\Sigma_k$、$V_k^T$。

### 直观理解

截断 SVD 本质上是在做这件事：

```
原始矩阵 A          ≈    词-主题矩阵 U     ×     主题强度 Σ     ×     主题-文档矩阵 V^T
(m 词 × n 文档)        (m 词 × k 主题)          (k 主题 × k 主题)       (k 主题 × n 文档)
```

比如：
- 主题1 = "金融领域"：`bank`、`money`、`stock` 权重高
- 主题2 = "自然地理"：`bank`、`river`、`water` 权重高
- 主题3 = "交通出行"：`car`、`drive`、`road` 权重高

通过 SVD，原本独立的词被组织到了少数几个"主题"之下，同义词会在同一个主题上有高权重。

### 为什么只保留前 k 个？

实际使用中，$k$ 通常取 $100 \sim 300$：

- 奇异值从大到小排列，前几个主题解释了数据中的大部分方差
- 后面的主题大多是噪声
- 截断后矩阵更紧凑，计算更快

这就是**降维**：从几万维的词汇空间，压缩到 $k$ 维的语义空间。

## 4.3 潜在语义分析（LSA）

### LSA = SVD + 截断

**LSA（Latent Semantic Analysis）** 就是把手里的词-文档矩阵做 SVD，然后截断到 $k$ 维。步骤如下：

```python
import numpy as np
from sklearn.decomposition import TruncatedSVD
from sklearn.feature_extraction.text import TfidfVectorizer

# ---------- 步骤 1：构建 TF-IDF 矩阵 ----------
docs = [
    "The faster Harry got to the store, the faster and faster Harry would get home.",
    "Harry is hairy and faster than Jill.",
    "Jill is not as hairy as Harry.",
    "The bank of the river was steep.",
    "Money in the bank earns interest."
]

vectorizer = TfidfVectorizer(stop_words='english')
A = vectorizer.fit_transform(docs)   # 形状: (5 文档, n 词汇)

# ---------- 步骤 2：Truncated SVD ----------
k = 2  # 降到 2 维（仅用于可视化，实际通常用 100~300）
svd = TruncatedSVD(n_components=k)
A_reduced = svd.fit_transform(A)      # 形状: (5 文档, 2 主题)

# A_reduced 就是文档在语义空间中的坐标
print("文档在语义空间中的坐标：")
print(A_reduced)
```

### 结果解读

降维后，每篇文档从一个几万维的稀疏向量，变成了一个 $k$ 维的稠密向量。

| 文档 | 主题1 权重 | 主题2 权重 | 语义解读 |
|------|-----------|-----------|---------|
| 文档1 (Harry faster...) | 0.85 | 0.12 | 偏向"Harry/速度"主题 |
| 文档4 (River bank...) | 0.15 | 0.78 | 偏向"自然/河岸"主题 |
| 文档5 (Money bank...) | 0.20 | 0.72 | 偏向"金融/银行"主题 |

> 注意：文档4 和 文档5 都包含 `bank`，但 SVD 把它们分到了不同的主题区域。这就是 LSA 的魔力——它通过**上下文**（周围的其他词）来区分 `bank` 的不同含义。

### 词的语义空间

如果我们对 $A^T$（文档-词矩阵）做 SVD，就能得到**词向量**：

```python
# 获取词汇表
terms = vectorizer.get_feature_names_out()

# 词-主题矩阵（U 矩阵）
U = svd.components_.T   # 形状: (n 词汇, k 主题)

# 查看 "bank" 和 "harry" 在语义空间中的坐标
bank_idx = np.where(terms == 'bank')[0][0]
harry_idx = np.where(terms == 'harry')[0][0]

print(f"bank: {U[bank_idx]}")
print(f"harry: {U[harry_idx]}")
```

## 4.4 LSA 的语义搜索

### 从关键词匹配到语义匹配

传统搜索：用户搜 "automobile"，只能找到包含这个词的文档。

LSA 搜索：把查询语句也映射到语义空间，找**距离最近**的文档。

```python
# 查询语句
query = "vehicle"
query_vec = vectorizer.transform([query])       # TF-IDF 向量
query_reduced = svd.transform(query_vec)        # 映射到语义空间

# 计算查询与所有文档的余弦相似度
from sklearn.metrics.pairwise import cosine_similarity
sims = cosine_similarity(query_reduced, A_reduced)

# 排序返回最相关文档
best_doc_idx = np.argmax(sims)
print(f"最相关文档: {docs[best_doc_idx]}")
```

**为什么有效？**
- 即使文档里没出现 "vehicle"，只要出现了 "car"、"drive"、"road" 等语义相近的词
- LSA 会把这些词映射到同一个主题区域
- 查询 "vehicle" 也会落在这个区域附近
- 于是就能"语义匹配"到相关文档

## 4.5 PCA：另一种降维视角

### PCA 与 SVD 的关系

**PCA（Principal Component Analysis）** 和 SVD 本质上是同一枚硬币的两面：

| | PCA | SVD |
|---|-----|-----|
| **数学对象** | 对协方差矩阵做特征分解 | 对原始矩阵做分解 |
| **输入要求** | 通常需要中心化（去均值） | 不需要中心化 |
| **输出** | 主成分方向 + 方差解释率 | 奇异向量 + 奇异值 |
| **关系** | 对中心化后的数据，PCA = SVD | SVD 更通用，不要求中心化 |

在 NLP 中，如果已经对 TF-IDF 矩阵做了归一化，**TruncatedSVD 和 PCA 的效果几乎一样**。sklearn 也提供了 `PCA` 类，但 `TruncatedSVD` 更适合稀疏矩阵。

### 选择指南

- **数据稀疏且维度高**（如 TF-IDF 矩阵）→ 用 `TruncatedSVD`
- **数据稠密且需要可视化**（如词嵌入降维到 2D/3D）→ 用 `PCA` 或 `t-SNE`


### 基于 PCA 的短消息语义分析
```python
from sklearn.decomposition import PCA

pca = pca.fit(tfidf_docs)
```

## 4.6 LDiA（潜在狄利克雷分布）
LDiA 假设每篇文档都由某个任意数量的主题混合（线性组合）而成，该数量是在开始训练 LDiA 模型时选择的。LDiA 还假设每个主题都可以用词的分布（词项频率）来表示。
文档中每个主题的概率或权重，以及某个词被分配到一个主题的概率，都假定一开始满足狄利克雷概率分布
### LdiA 思想
骰子的两轮投掷过程分别代表： 
 （1）生成文档的词的数量（泊松分布） 
 （2）文档中混合的主题的数量（狄利克雷分布）
设想的词袋生成机会在这些主题上迭代，并随机选择适合该主题的词，直到达到文档应该包含的词数量（见步骤 1）为止。

## 4.6 LSA 的局限

| 局限 | 说明 |
|------|------|
| **主题不可解释** | SVD 分解出来的"主题"是数学概念，不像 [[LDA]] 那样有明确的概率含义 |
| **线性假设** | SVD 只能捕捉线性关系，无法处理复杂的非线性语义 |
| **无法处理一词多义** | 虽然比原始 TF-IDF 好，但一个词的向量是所有含义的平均，不够精确 |
| **没有考虑词序** | 仍然是词袋模型的延伸，丢失了上下文信息 |

**下一代方案**：Word2Vec、GloVe、BERT 等**神经网络词向量**，通过预测上下文来学习词义，彻底解决了 LSA 的上述问题。

## 关键概念速查

| 术语         | 英文                           | 含义                                |
| ---------- | ---------------------------- | --------------------------------- |
| **词-文档矩阵** | Term-Document Matrix         | 行是词、列是文档、值是词频的稀疏矩阵                |
| **SVD**    | Singular Value Decomposition | 奇异值分解，将矩阵分解为 U·Σ·V^T              |
| **LSA**    | Latent Semantic Analysis     | 潜在语义分析，SVD + 截断降维                 |
| **潜在主题**   | Latent Topic                 | SVD 分解后得到的低维语义维度                  |
| **截断 SVD** | Truncated SVD                | 只保留前 k 个奇异值的 SVD                  |
| **奇异值**    | Singular Value               | Σ 对角线上的值，表示各主题的重要性                |
| **PCA**    | Principal Component Analysis | 主成分分析，另一种降维方法                     |
| **语义搜索**   | Semantic Search              | 不依赖关键词匹配，而是在语义空间中找相似              |
| **词嵌入**    | Word Embedding               | 低维稠密的词向量表示（LSA 是最早的 embedding 之一） |
| **降维**     | Dimensionality Reduction     | 将高维数据映射到低维空间，保留主要信息               |
