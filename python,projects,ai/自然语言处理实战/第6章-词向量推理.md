# 词向量推理（Word2Vec）

> 核心问题：如何用稠密低维向量捕捉词的语义？如何让"国王 - 男人 + 女人 ≈ 女王"这样的推理成为可能？

## 6.1 语义查询与类比

### 从搜索到推理

传统搜索引擎面对模糊查询时往往无能为力：

> She invented something to do with physics in Europe in the early 20th century.

这类查询没有直接的关键词匹配，但 Word2Vec 让"语义计算"成为可能。

**核心假设**：语义可以通过向量的**线性运算**来组合。

```
"woman" + "Europe" + "physics" + "scientist" + "famous"
  → 指向 Marie_Curie 的方向
```

### 类比问题

> Who is to nuclear physics what Louis Pasteur is to germs?

**解法**：向量线性运算

```python
result = word_vectors.most_similar(
    positive=['Louis_Pasteur', 'nuclear', 'physics'],
    negative=['germs'],
    topn=1
)
# 指向: Marie_Curie
```

**原理**：`Louis_Pasteur - germs` 提取出"科学家"的语义方向，`+ nuclear + physics` 把这个方向应用到核物理领域。

## 6.2 词向量（Word2Vec）

### 什么是 Word2Vec？

**Word2Vec** 是 Mikolov 于 2013 年提出的词嵌入方法。它通过训练神经网络预测词的上下文，让语义相近的词在向量空间中距离相近。

**关键特性**：
- **无监督学习**：只需要未标注的语料库
- **稠密低维**：通常 100~300 维，远小于词汇表大小
- **语义捕获**：向量距离 ≈ 语义相似度

### 两种训练方法

| | Skip-gram | CBOW（连续词袋）|
|---|-----------|----------------|
| **输入** | 中心词 | 周围词的 multi-hot vector |
| **输出** | 预测周围词 | 预测中心词 |
| **适用场景** | 小型语料、罕见词 | 大型语料、常见词 |
| **训练速度** | 较慢（样本多） | 较快 |
| **罕见词效果** | 更好 | 一般 |

#### Skip-gram 详解

```
句子: "the cat sat on the mat"
窗口大小 = 2

中心词: "sat"
上下文: ["the", "cat", "on", "the"]

输入: "sat" 的 one-hot
输出: 预测 "the", "cat", "on", "the" 的概率

训练完成后：
  隐藏层权重矩阵 W (V × embed_dim) 就是词嵌入表
  词向量 = one-hot(词) · W
```

#### CBOW 详解

```
中心词: "sat"
上下文: ["the", "cat", "on", "the"]

输入: "the", "cat", "on", "the" 的向量平均
输出: 预测 "sat"
```

> 💡 **工程建议**：语料小、罕见词多 → 用 Skip-gram；语料大、追求速度 → 用 CBOW。

## 6.3 Word2Vec 训练技巧

### 6.3.1 高频 2-gram 合并

**问题**：很多固定搭配（如 "New York", "San Francisco"）作为两个独立词相互预测没有意义。

**解决方案**：将高频共现的 n-gram 作为**单一词项**加入词汇表。

**打分函数**：

$$
score(w_i, w_j) = \frac{count(w_i, w_j) - \delta}{count(w_i) \times count(w_j)}
$$

其中 $\delta$ 是折扣系数。分数高于阈值时，将 $w_i\_w_j$ 作为新词项。

### 6.3.2 高频词条降采样

**问题**：`the`、`a` 等高频词出现太多，但信息量少，会主导训练。

**Mikolov 的降采样公式**：

$$
P(w_i) = 1 - \sqrt{\frac{t}{f(w_i)}}
$$

其中：
- $f(w_i)$ = 词 $w_i$ 在语料库中的出现频率
- $t$ = 阈值（通常 $10^{-5} \sim 10^{-6}$）
- $P(w_i)$ = 该词被**保留**的概率（不是跳过概率）

**解读**：
- 频率远高于阈值的词 → 保留概率低 → 大量丢弃
- 频率接近阈值的词 → 保留概率 ≈ 0 → 几乎全丢弃
- 频率低于阈值的词 → 保留概率 > 0 → 保留

> ⚠️ **注意**：原书中公式写的是保留概率，但描述时说"有这么大的概率会跳过这个词"，实际代码实现需确认是保留还是跳过。

### 6.3.3 负采样

**问题**：Softmax 要计算整个词汇表（10万+词），每次更新太慢。

**核心思想**：把多分类变成少量二分类。

```
原始 Softmax:
  问: "sat" 的上下文是 "cat" 吗？在所有 10 万个词中排第几？
  → 计算量 O(V)

负采样:
  问: "cat" 是 "sat" 的真实上下文，还是随机抽的噪声？
  → 只算 1 个正样本 + k 个负样本
  → 计算量 O(k+1)，k 通常 5~20
```

**损失函数**：

$$
\text{Loss} = \underbrace{\log \sigma(u_o^T v_c)}_{\text{正样本}} + \underbrace{\sum_{i=1}^{k} \log \sigma(-u_{n_i}^T v_c)}_{\text{k 个负样本}}
$$

**负样本采样**：按词频的 $3/4$ 次幂采样，压制高频词：

$$
P_n(w) = \frac{f(w)^{3/4}}{\sum_{w'} f(w')^{3/4}}
$$

> 详见 [[负采样|负采样详解]]

## 6.4 GloVe

### 核心思想

**GloVe = Global Vectors**，结合了两派优点：
- **LSA** 的全局统计（共现矩阵）
- **Word2Vec** 的局部预测（上下文窗口）

### 与 Word2Vec 的对比

| | Word2Vec | GloVe |
|---|----------|-------|
| **利用信息** | 局部上下文窗口 | 全局词共现矩阵 |
| **训练方式** | 滑动窗口逐样本 | 直接对共现矩阵分解 |
| **速度** | 快 | 中等 |
| **内存** | 低 | 高（需存共现矩阵）|
| **罕见词** | 较差 | 较好 |
| **可解释性** | 弱 | 较强 |

### 损失函数

GloVe 让词向量内积逼近共现概率的对数：

$$
J = \sum_{i,j=1}^{V} f(X_{ij}) \left( u_i^T v_j + b_i + b_j - \log X_{ij} \right)^2
$$

其中：
- $X_{ij}$ = 词 $i$ 和词 $j$ 的共现次数
- $f(X_{ij})$ = 权重函数（避免 $\log 0$）
- $b_i, b_j$ = 偏置项

### 优点

- 训练过程更快
- 对硬件的有效利用
- 对数据的有效利用
- 相同训练次数的精确率更高

## 6.5 fastText

### 核心思想

**词 = 子词（subword）的集合**

与 Word2Vec 把词当原子不同，fastText 将词拆成 n-gram 子词，词向量是所有子词向量之和。

```
"where" → "<where>"

3-gram: <wh, whe, her, ere, re>
4-gram: <whe, wher, here, ere>

vector("where") = vector(<wh) + vector(whe) + vector(her) + ... + vector(here)
```

### 三大优势

| 优势 | 说明 | 示例 |
|------|------|------|
| **解决 OOV** | 没见过的词也能表示 | `computers` = `computer` 的子词 + `s` 的子词 |
| **捕捉形态** | 复数、时态、派生词自动关联 | `run` ≈ `running` ≈ `ran` |
| **跨语言** | 不同语言共享子词 | 英语 `university` 和西班牙语 `universidad` 共享 `univers` |

### 训练方式

和 Word2Vec 的 Skip-gram 几乎一样，只是**输入变成子词向量之和**：

```
Word2Vec Skip-gram:
  输入: 中心词 "apple" 的 one-hot → 词向量
  
fastText Skip-gram:
  输入: "apple" 的所有子词向量之和
        = vector(<ap) + vector(app) + vector(ppl) + vector(ple) + vector(le>)
```

### 代码

```python
from gensim.models import FastText

# 训练
model = FastText(sentences, vector_size=100, window=3, min_count=1)

# OOV 也能处理！
model.wv['computers']   # ✅ 自动用子词组合
model.wv['runing']      # ✅ 拼写错误也能给向量
```

## 6.6 Doc2Vec

### 核心思想

不是简单地把词向量相加，而是给每个文档学一个**独立的向量**。

```
Word2Vec 的输入:
  [词向量矩阵 W] × one-hot(词)

Doc2Vec 的输入:
  [词向量矩阵 W] × one-hot(词) + [文档向量矩阵 D] × one-hot(文档ID)
```

### 两种变体

| 变体 | 简称 | 训练方式 |
|------|------|---------|
| **Distributed Memory** | **PV-DM** | 文档向量 + 上下文词向量 → 预测中心词（类似 CBOW）|
| **Distributed Bag of Words** | **PV-DBOW** | 只用文档向量 → 预测上下文词（类似 Skip-gram）|

### 与词向量相加的区别

| | 词向量相加 | Doc2Vec |
|---|-----------|---------|
| **文档表示** | 词向量的平均/求和 | **独立训练的文档向量** |
| **词序信息** | ❌ 完全丢失 | ✅ 训练时隐式捕获 |
| **文档语义** | 只是词的堆砌 | 学到文档特有的主题 |
| **相似文档** | 有共同词就相似 | 语义主题相似才相似 |

### 代码

```python
from gensim.models.doc2vec import Doc2Vec, TaggedDocument

# 给文档打标签
tagged_docs = [
    TaggedDocument(words=["the", "cat", "sat"], tags=["doc_1"]),
    TaggedDocument(words=["dog", "ran", "park"], tags=["doc_2"]),
]

# 训练
model = Doc2Vec(tagged_docs, vector_size=100, window=2, min_count=1)

# 获取文档向量
doc_vec = model.dv['doc_1']

# 找相似文档
similar = model.dv.most_similar('doc_1')

# 推断新文档向量
new_vec = model.infer_vector(["machine", "learning", "deep"])
```

## 6.7 使用 gensim 加载与查询词向量

### 加载预训练模型

```python
from gensim.models.keyedvectors import KeyedVectors

# 加载 GoogleNews 词向量
word_vectors = KeyedVectors.load_word2vec_format(
    "/Users/maverick/Downloads/GoogleNews-vectors-negative300.bin.gz",
    binary=True
)
```

### 常用查询方法

#### 1. 获取词向量

```python
vector = word_vectors['computer']
print(vector.shape)  # (300,)
```

#### 2. 检查词是否存在

```python
if 'computer' in word_vectors:
    vec = word_vectors['computer']
else:
    print("词不在词汇表中")
```

#### 3. 找最相似的词

```python
similar = word_vectors.most_similar('king', topn=5)
# [('king', 1.0), ('queen', 0.71), ('monarch', 0.62), ...]
```

#### 4. 计算相似度

```python
sim = word_vectors.similarity('cat', 'dog')
print(sim)  # 0.88
```

#### 5. 类比推理

```python
# king - man + woman = ?
result = word_vectors.most_similar(
    positive=['king', 'woman'],
    negative=['man'],
    topn=1
)
# [('queen', 0.76)]
```

#### 6. 找出不匹配的词

```python
word = word_vectors.doesnt_match(['apple', 'banana', 'computer', 'orange'])
print(word)  # 'computer'
```

### 训练自己的 Word2Vec

```python
from gensim.models.word2vec import Word2Vec

# 训练
model = Word2Vec(sentences, vector_size=100, window=5, min_count=1)

# 保存
model.save("my_word2vec.model")
```

## 6.8 非自然词

Word2Vec 甚至可以从**非自然语言**中提取信息：
- 象形语言（如古埃及文字）
- 密码/编码文本
- 基因序列
- 任何有"共现模式"的符号系统

**核心原因**：只要符号之间存在**上下文共现关系**，Word2Vec 就能学到它们的向量表示。

## 关键概念速查

### 6.1 ~ 6.2 Word2Vec 基础

| 术语 | 英文 | 含义 |
|------|------|------|
| 词嵌入 | Word Embedding | 用稠密低维向量表示词语义 |
| Word2Vec | Word2Vec | Mikolov 提出的词向量训练框架 |
| Skip-gram | Skip-gram | 用中心词预测周围词 |
| CBOW | Continuous Bag of Words | 用周围词预测中心词 |
| 负采样 | Negative Sampling | 用少量负样本替代全词汇表 Softmax |
| 降采样 | Subsampling | 减少高频词的训练样本 |
| OOV | Out-Of-Vocabulary | 词汇表外的词 |

### 6.3 训练技巧

| 术语 | 英文 | 含义 |
|------|------|------|
| 2-gram 合并 | Phrase Detection | 将高频共现词对作为单一词项 |
| 子词 | Subword | 词的 n-gram 片段 |
| 上下文窗口 | Context Window | 中心词两侧考虑的词数 |

### 6.4 ~ 6.6 其他词向量方法

| 术语 | 英文 | 含义 |
