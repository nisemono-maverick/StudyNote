# 分词（Tokenization）

> 核心问题：如何把非结构化的文本，转化为机器能处理的结构化向量？

---

## 2.1 分词面临的挑战

### 词干还原（Stemming）
- 动词 ing → 原形：`running` → `run`
- 名词复数 → 单数：`cats` → `cat`
- ⚠️ 注意：不是所有语言都像英语这么规则，中文不存在这个问题

### 其他挑战（待补充）
- 大小写歧义：`US`（美国）vs `us`
- 标点粘连：`26.`、`don't`
- 社交媒体非标准文本：表情符号、缩写、拼写错误

---

## 2.2 从文本到向量

分词只是第一步，最终目的是把文本变成**向量**（Vector）。

### 2.2.0 基础分词方法

#### 方法 1：按空格切分（最简单）
```python
sentence = """Thomas Jefferson began building Monticello at the age of 26."""
tokens = sentence.split()
# ['Thomas', 'Jefferson', 'began', 'building', 'Monticello', 'at', 'the', 'age', 'of', '26.']
```
- ❌ 问题：标点会粘在词上（如 `26.`）

#### 方法 2：正则表达式分词
```python
import re
pattern = re.compile(r"([-\s.,;!?])+")
tokens = pattern.split(sentence)
tokens = [x for x in tokens if x and x not in '- \t\n.,;!?']
# ['Thomas', 'Jefferson', 'began', 'building', 'Monticello', 'at', 'the', 'age', 'of', '26']
```
- ✅ 标点被正确分离

---

### 2.2.1 向量表示方法对比

| 方法 | 保留词序 | 维度特点 | 核心思想 |
|------|---------|---------|---------|
| **独热向量** | ✅ 完全保留 | 极高（= 词汇表大小） | 每个词一个维度 |
| **词袋向量** | ❌ 完全丢弃 | 中等（= 词汇表大小） | 累加词频 |
| **n-gram** | 局部保留 | 更高（组合爆炸） | 用相邻词组捕捉局部语义 |

#### ① 独热向量（One-Hot）

**原理**：为词汇表中每个词分配一个独立维度，该词出现的位置记为 1，其余为 0。

```python
import numpy as np

token_sequence = str.split(sentence)
vocab = sorted(set(token_sequence))
num_tokens = len(token_sequence)
vocab_size = len(vocab)

onehot_vectors = np.zeros((num_tokens, vocab_size), dtype=int)
for i, token in enumerate(token_sequence):
    token_index = vocab.index(token)
    onehot_vectors[i, token_index] = 1
```

用 pandas 查看：
```python
import pandas as pd
pd.DataFrame(onehot_vectors, columns=vocab)
```

**优点**：信息无丢失，每个词完全独立可区分  
**缺点**：
- 维度灾难：英语常用词 2万+，加专有名词可达百万级
- 矩阵极度稀疏（Sparse）
- 每个新文本都要重建矩阵

#### ② 词袋向量（Bag of Words, BOW）

**原理**：假设"词序不重要"，将一段文本中所有词的独热向量**相加**（或取或）。

```python
sentence_bow = {}
for token in sentence.split():
    sentence_bow[token] = 1
sorted(sentence_bow.items())
```

**频率版**（更常见）：
```python
from collections import Counter
sentence_bow = Counter(sentence.split())
```

**优点**：维度固定，忽略词序后大大简化  
**缺点**：
- 丢失了否定、语序等关键语义（如 `not good` vs `good not bad`）
- 无法处理一词多义

#### ③ n-gram

**定义**：由 n 个连续词组成的序列。
- 1-gram（unigram）：`Thomas`, `Jefferson`...
- 2-gram（bigram）：`Thomas Jefferson`, `Jefferson began`...
- 3-gram（trigram）：`Thomas Jefferson began`...

**为什么需要 n-gram？**
- 词袋模型丢失了词序，`not` 和相邻词分离后，否定含义可能漂移
- n-gram 能捕捉**局部搭配**和**短语边界**

```python
from nltk.util import ngrams
list(ngrams(tokens, 2))
# [('Thomas', 'Jefferson'), ('Jefferson', 'began'), ...]
```

**代价**：维度进一步膨胀，2-gram 的词汇表是原词汇表的平方级。

### 2.2.2 度量文本相似度

#### 点积（Dot Product）
- 计算两个词袋向量之间的重合度
- 即两个句子**共同出现的词语个数**
- ⚠️ 受文本长度影响大（长文本天然更容易有高点积）

#### 余弦相似度（待补充）
- 归一化的点积，消除长度影响
- 公式：$\cos(\theta) = \frac{A \cdot B}{\|A\| \|B\|}$

### 2.2.4 停用词（Stop Words）

**定义**：出现频率极高、但承载语义信息极少的词（如 `a`, `an`, `the`, `is`）。

**处理方式**：
- 通常维护一个停用词表，分词后过滤
- NLTK、spaCy 都有内置停用词表

**⚠️ 风险**：
- 盲目剔除会丢失信息：*The Shining*（电影名）变成 *Shining*，失去特指含义
- 否定词（`not`, `no`）有时在停用词表中，去除会毁掉情感分析

**决策原则**：根据下游任务决定是否去除，不要一刀切。

### 2.2.5 词汇表归一化

#### 大小写转换（Case Normalization）

**目的**：减小词汇表规模（`The` 和 `the` 合并）。

**风险**：
- 专有名词含义改变：`US`（美国）→ `us`（我们），`Doctor`（头衔）→ `doctor`（职业）
- 句首专有名词被误处理：`Apple`（苹果公司）→ `apple`（水果）
- 驼峰命名丢失信息：`WordTokenization` → `wordtokenization`

**折中方案**：只将句首单词小写，其余保留原样。  
**现代趋势**：很多 NLP 流水线**不再做大小写归一化**，尤其是预训练语言模型（如 BERT）保留大小写信息对下游任务有帮助。

#### 词干还原（Stemming）

**原理**：基于规则的**粗暴截断**，去掉词缀（前缀/后缀）得到词干。结果**不一定是真实存在的单词**。

```python
from nltk.stem.porter import PorterStemmer

stemmer = PorterStemmer()
words = "dish washer's washed dishes running runs automate".split()
result = ' '.join([stemmer.stem(w).strip("'") for w in words])
print(result)
# dish washer wash dish run run automat
```

| 变形词 | 词干 | 评价 |
|--------|------|------|
| `running` | `run` | ✅ 正确 |
| `automate` | `automat` | ⚠️ 过度截断 |
| `nationalization` | `nation` | ⚠️ 丢失后缀信息 |

**主流算法**：
- **Porter Stemmer**：最经典，规则简单、速度快，但过度截断问题明显
- **Snowball Stemmer**：Porter 的改进版，支持多语言，更精确
- **Lancaster Stemmer**：更激进，截断更狠，信息损失更大

**优点**：
- 速度极快，无需词典
- 显著压缩词汇表规模

**⚠️ 风险**：
- **过度截断**：`university` → `univers`，`automate` → `automat`（失去语义）
- 返回结果可能是**伪词**（不是词典里的真实单词）
- 会大幅降低搜索引擎的**精确率**（返回大量不相关文档）
- 大部分搜索引擎支持加双引号 `"exact word"` 来关闭词干还原，进行精确匹配

#### 词形归并（Lemmatization）

**原理**：基于**词典和词性（POS）标签**，将变形词还原为规范的**词元（Lemma）**。结果是真实存在的单词原型。

```python
from nltk.stem import WordNetLemmatizer

lemmatizer = WordNetLemmatizer()

# 不指定词性时，默认按名词处理
lemmatizer.lemmatize("running")       # 'running' （被当成名词）
lemmatizer.lemmatize("better")        # 'better'  （被当成名词）

# 指定词性后结果更精确
lemmatizer.lemmatize("running", pos='v')   # 'run'  （动词）
lemmatizer.lemmatize("better", pos='a')    # 'good' （形容词）
```

**为什么需要 POS 标签？**
同一个拼写可能是不同词性，lemma 也不同：

| 原词 | 词性 | Lemma |
|------|------|-------|
| `saw` | 动词 (v) | `see` |
| `saw` | 名词 (n) | `saw`（锯子）|
| `leaves` | 动词 (v) | `leave` |
| `leaves` | 名词 (n) | `leaf` |

**优点**：
- 结果是**合法词汇**，可直接用于词典查询
- 语义保留更完整，错误率低于词干还原

**缺点**：
- 速度较慢（需要查词典和词性标注）
- 需要准确的 POS 标签才能达到最佳效果
- 词汇表外（OOV）的新词无法处理

#### 对比：Stemming vs Lemmatization

| 维度 | 词干还原 (Stemming) | 词形归并 (Lemmatization) |
|------|---------------------|--------------------------|
| **核心思想** | 基于规则截断词缀 | 基于词典还原为规范形式 |
| **结果性质** | 可能是伪词 | 必须是真实单词 |
| **速度** | 极快 | 较慢（需查词典） |
| **准确度** | 低，容易过度截断 | 高，语义保留完整 |
| **POS 依赖** | 不需要 | 需要（或受益于）POS 标签 |
| **代表工具** | PorterStemmer, Snowball | WordNetLemmatizer, spaCy |


#### 使用场景与决策建议

**词干还原更适合**：
- 对速度要求极高的海量文本处理
- 信息检索（IR）中追求**高召回率**的场景
- 早期/传统 NLP 流水线

**词形归并更适合**：
- 需要高语义精度的任务（文本分类、情感分析、机器翻译）
- 下游任务依赖**真实词汇**（如知识图谱、词典匹配）
- 现代 NLP 预处理

**混合策略（工业界常用）**：
> 追求精确率的应用，往往**先基于原始词进行精确匹配**；若失败，再对归一化后的词进行模糊匹配，寻找可能的结果。

- 搜索系统：先用原始词查，再用 lemma/stem 扩展召回
- 这样既能保证精确率，又能提升召回率

## 2.3 情感
- 情感分析：每个词条中包含的情感。
- 情感分析的应用
	- 收集用户反馈信息，然后对反馈内容进行分析
	- 过滤垃圾邮件或钓鱼信息
- 两种情感分析的方法
	- 基于规则的算法，规则由人编写
		- 基于用户设计的启发式规则来度量文本的情感，一个常用的作法是关键词索引（VADER 算法）
	- 基于机器学习的算法，从数据量中学习而得模型
		- 利用一系列标注语句或文档来训练机器学习模型以产生规则（朴素贝叶斯）

### 2.3.1 VADER：一个基于规则的情感分析器
- 使用 vaderSentiment 包进行情感分析
```python
from vaderSentiment.vaderSentiment import SentimentIntensityAnalyzer
sa = SentimentIntensityAnalyzer()

corpus = [
    "Absolutely perfect! Love it! :-) :-) :-)",
    "Horrible! Completely useless. :(",
    "It was OK. Some good and some bad things."
]
for doc in corpus:
    scores = sa.polarity_scores(doc)
    print('{:+}: {}'.format(scores['compound'], doc))

# +0.9428: Absolutely perfect! Love it! :-) :-) :-)
# -0.8768: Horrible! Completely useless. :(
# -0.1531: It was OK. Some good and some bad things.
```
- vader 的唯一不足在于，它只关注其词库中有的词条，而非文档中的所有词
- 如果用户对于他要处理的语言完全不了解的话，这套方法完全不可用
