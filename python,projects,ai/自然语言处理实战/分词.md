## 2.1 分词面临的挑战

### 词干还原（Stemming）
- 动词 ing → 原形：`running` → `run`
- 名词复数 → 单数：`cats` → `cat`
- ⚠️ 注意：不是所有语言都像英语这么规则，中文不存在这个问题

### 其他挑战（待补充）
- 大小写歧义：`US`（美国）vs `us`
- 标点粘连：`26.`、`don't`
- 社交媒体非标准文本：表情符号、缩写、拼写错误

## 2.2 从文本到向量

分词只是第一步，最终目的是把文本变成**向量**（Vector）。

### 2.2.0 基础分词方法

#### 方法 1：按空格切分（最简单）
```python
sentence = """Thomas Jefferson began building Monticello at the age of 26."""
tokens = sentence.split()
# ['Thomas', 'Jefferson', 'began', 'building', 'Monticello', 'at', 'the', 'age', 'of', '26.']
```
- ❌ 问题：标点会粘在词上（如 `26.`）

#### 方法 2：正则表达式分词
```python
import re
pattern = re.compile(r"([-\s.,;!?])+")
tokens = pattern.split(sentence)
tokens = [x for x in tokens if x and x not in '- \t\n.,;!?']
# ['Thomas', 'Jefferson', 'began', 'building', 'Monticello', 'at', 'the', 'age', 'of', '26']
```
- ✅ 标点被正确分离

---

### 2.2.1 向量表示方法对比

| 方法 | 保留词序 | 维度特点 | 核心思想 |
|------|---------|---------|---------|
| **独热向量** | ✅ 完全保留 | 极高（= 词汇表大小） | 每个词一个维度 |
| **词袋向量** | ❌ 完全丢弃 | 中等（= 词汇表大小） | 累加词频 |
| **n-gram** | 局部保留 | 更高（组合爆炸） | 用相邻词组捕捉局部语义 |

#### ① 独热向量（One-Hot）

**原理**：为词汇表中每个词分配一个独立维度，该词出现的位置记为 1，其余为 0。

```python
import numpy as np

token_sequence = str.split(sentence)
vocab = sorted(set(token_sequence))
num_tokens = len(token_sequence)
vocab_size = len(vocab)

onehot_vectors = np.zeros((num_tokens, vocab_size), dtype=int)
for i, token in enumerate(token_sequence):
    token_index = vocab.index(token)
    onehot_vectors[i, token_index] = 1
```

用 pandas 查看：
```python
import pandas as pd
pd.DataFrame(onehot_vectors, columns=vocab)
```

**优点**：信息无丢失，每个词完全独立可区分  
**缺点**：
- 维度灾难：英语常用词 2万+，加专有名词可达百万级
- 矩阵极度稀疏（Sparse）
- 每个新文本都要重建矩阵

#### ② 词袋向量（Bag of Words, BOW）

**原理**：假设"词序不重要"，将一段文本中所有词的独热向量**相加**（或取或）。

```python
sentence_bow = {}
for token in sentence.split():
    sentence_bow[token] = 1
sorted(sentence_bow.items())
```

**频率版**（更常见）：
```python
from collections import Counter
sentence_bow = Counter(sentence.split())
```

**优点**：维度固定，忽略词序后大大简化  
**缺点**：
- 丢失了否定、语序等关键语义（如 `not good` vs `good not bad`）
- 无法处理一词多义

#### ③ n-gram

**定义**：由 n 个连续词组成的序列。
- 1-gram（unigram）：`Thomas`, `Jefferson`...
- 2-gram（bigram）：`Thomas Jefferson`, `Jefferson began`...
- 3-gram（trigram）：`Thomas Jefferson began`...

**为什么需要 n-gram？**
- 词袋模型丢失了词序，`not` 和相邻词分离后，否定含义可能漂移
- n-gram 能捕捉**局部搭配**和**短语边界**

```python
from nltk.util import ngrams
list(ngrams(tokens, 2))
# [('Thomas', 'Jefferson'), ('Jefferson', 'began'), ...]
```

**代价**：维度进一步膨胀，2-gram 的词汇表是原词汇表的平方级。

### 2.2.2 度量文本相似度

#### 点积（Dot Product）
- 计算两个词袋向量之间的重合度
- 即两个句子**共同出现的词语个数**
- ⚠️ 受文本长度影响大（长文本天然更容易有高点积）

#### 余弦相似度（待补充）
- 归一化的点积，消除长度影响
- 公式：$\cos(\theta) = \frac{A \cdot B}{\|A\| \|B\|}$

### 2.2.4 停用词（Stop Words）

**定义**：出现频率极高、但承载语义信息极少的词（如 `a`, `an`, `the`, `is`）。

**处理方式**：
- 通常维护一个停用词表，分词后过滤
- NLTK、spaCy 都有内置停用词表

**⚠️ 风险**：
- 盲目剔除会丢失信息：*The Shining*（电影名）变成 *Shining*，失去特指含义
- 否定词（`not`, `no`）有时在停用词表中，去除会毁掉情感分析

**决策原则**：根据下游任务决定是否去除，不要一刀切。

---

### 2.2.5 词汇表归一化

#### 大小写转换（Case Normalization）

**目的**：减小词汇表规模（`The` 和 `the` 合并）。

**风险**：
- 专有名词含义改变：`US`（美国）→ `us`（我们），`Doctor`（头衔）→ `doctor`（职业）
- 句首专有名词被误处理：`Apple`（苹果公司）→ `apple`（水果）
- 驼峰命名丢失信息：`WordTokenization` → `wordtokenization`

**折中方案**：只将句首单词小写，其余保留原样。  
**现代趋势**：很多 NLP 流水线**不再做大小写归一化**，尤其是预训练语言模型（如 BERT）保留大小写信息对下游任务有帮助。

## 关键概念速查

| 术语 | 英文 | 含义 |
|------|------|------|
| 分词器 | Tokenizer / Lexer | 将文本切分为词或符号的扫描器 |
| 词汇表 | Vocabulary | 所有不重复词条的集合 |
| 词干还原 | Stemming | 去除词缀得到词根（粗略） |
| 词形归并 | Lemmatization | 基于词典还原词形（精确，保留词性） |
| 词条 | Token | 分词后的最小单位 |
| n-gram | n-gram | n 个连续词组成的序列 |
