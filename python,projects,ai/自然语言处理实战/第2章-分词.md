# 分词（Tokenization）

> 核心问题：如何把非结构化的文本，转化为机器能处理的结构化向量？

---

## 2.1 分词面临的挑战

### 词干还原（Stemming）
- 动词 ing → 原形：`running` → `run`
- 名词复数 → 单数：`cats` → `cat`
- ⚠️ 注意：不是所有语言都像英语这么规则，中文不存在这个问题

### 其他挑战（待补充）
- 大小写歧义：`US`（美国）vs `us`
- 标点粘连：`26.`、`don't`
- 社交媒体非标准文本：表情符号、缩写、拼写错误

---

## 2.2 从文本到向量

分词只是第一步，最终目的是把文本变成**向量**（Vector）。

### 2.2.0 基础分词方法

#### 方法 1：按空格切分（最简单）
```python
sentence = """Thomas Jefferson began building Monticello at the age of 26."""
tokens = sentence.split()
# ['Thomas', 'Jefferson', 'began', 'building', 'Monticello', 'at', 'the', 'age', 'of', '26.']
```
- ❌ 问题：标点会粘在词上（如 `26.`）

#### 方法 2：正则表达式分词
```python
import re
pattern = re.compile(r"([-\s.,;!?])+")
tokens = pattern.split(sentence)
tokens = [x for x in tokens if x and x not in '- \t\n.,;!?']
# ['Thomas', 'Jefferson', 'began', 'building', 'Monticello', 'at', 'the', 'age', 'of', '26']
```
- ✅ 标点被正确分离

---

### 2.2.1 向量表示方法对比

| 方法 | 保留词序 | 维度特点 | 核心思想 |
|------|---------|---------|---------|
| **独热向量** | ✅ 完全保留 | 极高（= 词汇表大小） | 每个词一个维度 |
| **词袋向量** | ❌ 完全丢弃 | 中等（= 词汇表大小） | 累加词频 |
| **n-gram** | 局部保留 | 更高（组合爆炸） | 用相邻词组捕捉局部语义 |

#### ① 独热向量（One-Hot）

**原理**：为词汇表中每个词分配一个独立维度，该词出现的位置记为 1，其余为 0。

```python
import numpy as np

token_sequence = str.split(sentence)
vocab = sorted(set(token_sequence))
num_tokens = len(token_sequence)
vocab_size = len(vocab)

onehot_vectors = np.zeros((num_tokens, vocab_size), dtype=int)
for i, token in enumerate(token_sequence):
    token_index = vocab.index(token)
    onehot_vectors[i, token_index] = 1
```

用 pandas 查看：
```python
import pandas as pd
pd.DataFrame(onehot_vectors, columns=vocab)
```

**优点**：信息无丢失，每个词完全独立可区分  
**缺点**：
- 维度灾难：英语常用词 2万+，加专有名词可达百万级
- 矩阵极度稀疏（Sparse）
- 每个新文本都要重建矩阵

#### ② 词袋向量（Bag of Words, BOW）

**原理**：假设"词序不重要"，将一段文本中所有词的独热向量**相加**（或取或）。

```python
sentence_bow = {}
for token in sentence.split():
    sentence_bow[token] = 1
sorted(sentence_bow.items())
```

**频率版**（更常见）：
```python
from collections import Counter
sentence_bow = Counter(sentence.split())
```

**优点**：维度固定，忽略词序后大大简化  
**缺点**：
- 丢失了否定、语序等关键语义（如 `not good` vs `good not bad`）
- 无法处理一词多义

#### ③ n-gram

**定义**：由 n 个连续词组成的序列。
- 1-gram（unigram）：`Thomas`, `Jefferson`...
- 2-gram（bigram）：`Thomas Jefferson`, `Jefferson began`...
- 3-gram（trigram）：`Thomas Jefferson began`...

**为什么需要 n-gram？**
- 词袋模型丢失了词序，`not` 和相邻词分离后，否定含义可能漂移
- n-gram 能捕捉**局部搭配**和**短语边界**

```python
from nltk.util import ngrams
list(ngrams(tokens, 2))
# [('Thomas', 'Jefferson'), ('Jefferson', 'began'), ...]
```

**代价**：维度进一步膨胀，2-gram 的词汇表是原词汇表的平方级。

---

### 2.2.2 度量文本相似度

#### 点积（Dot Product）
- 计算两个词袋向量之间的重合度
- 即两个句子**共同出现的词语个数**
- ⚠️ 受文本长度影响大（长文本天然更容易有高点积）

#### 余弦相似度（待补充）
- 归一化的点积，消除长度影响
- 公式：$\cos(\theta) = \frac{A \cdot B}{\|A\| \|B\|}$

---

### 2.2.4 停用词（Stop Words）

**定义**：出现频率极高、但承载语义信息极少的词（如 `a`, `an`, `the`, `is`）。

**处理方式**：
- 通常维护一个停用词表，分词后过滤
- NLTK、spaCy 都有内置停用词表

**⚠️ 风险**：
- 盲目剔除会丢失信息：*The Shining*（电影名）变成 *Shining*，失去特指含义
- 否定词（`not`, `no`）有时在停用词表中，去除会毁掉情感分析

**决策原则**：根据下游任务决定是否去除，不要一刀切。

---

### 2.2.5 词汇表归一化

#### 大小写转换（Case Normalization）

**目的**：减小词汇表规模（`The` 和 `the` 合并）。

**风险**：
- 专有名词含义改变：`US`（美国）→ `us`（我们），`Doctor`（头衔）→ `doctor`（职业）
- 句首专有名词被误处理：`Apple`（苹果公司）→ `apple`（水果）
- 驼峰命名丢失信息：`WordTokenization` → `wordtokenization`

**折中方案**：只将句首单词小写，其余保留原样。  
**现代趋势**：很多 NLP 流水线**不再做大小写归一化**，尤其是预训练语言模型（如 BERT）保留大小写信息对下游任务有帮助。

#### 词干还原（Stemming）

**原理**：基于规则的**粗暴截断**，去掉词缀（前缀/后缀）得到词干。结果**不一定是真实存在的单词**。

```python
from nltk.stem.porter import PorterStemmer

stemmer = PorterStemmer()
words = "dish washer's washed dishes running runs automate".split()
result = ' '.join([stemmer.stem(w).strip("'") for w in words])
print(result)
# dish washer wash dish run run automat
```

| 变形词 | 词干 | 评价 |
|--------|------|------|
| `running` | `run` | ✅ 正确 |
| `automate` | `automat` | ⚠️ 过度截断 |
| `nationalization` | `nation` | ⚠️ 丢失后缀信息 |

**主流算法**：
- **Porter Stemmer**：最经典，规则简单、速度快，但过度截断问题明显
- **Snowball Stemmer**：Porter 的改进版，支持多语言，更精确
- **Lancaster Stemmer**：更激进，截断更狠，信息损失更大

**优点**：
- 速度极快，无需词典
- 显著压缩词汇表规模

**⚠️ 风险**：
- **过度截断**：`university` → `univers`，`automate` → `automat`（失去语义）
- 返回结果可能是**伪词**（不是词典里的真实单词）
- 会大幅降低搜索引擎的**精确率**（返回大量不相关文档）
- 大部分搜索引擎支持加双引号 `"exact word"` 来关闭词干还原，进行精确匹配

#### 词形归并（Lemmatization）

**原理**：基于**词典和词性（POS）标签**，将变形词还原为规范的**词元（Lemma）**。结果是真实存在的单词原型。

```python
from nltk.stem import WordNetLemmatizer

lemmatizer = WordNetLemmatizer()

# 不指定词性时，默认按名词处理
lemmatizer.lemmatize("running")       # 'running' （被当成名词）
lemmatizer.lemmatize("better")        # 'better'  （被当成名词）

# 指定词性后结果更精确
lemmatizer.lemmatize("running", pos='v')   # 'run'  （动词）
lemmatizer.lemmatize("better", pos='a')    # 'good' （形容词）
```

**为什么需要 POS 标签？**
同一个拼写可能是不同词性，lemma 也不同：

| 原词 | 词性 | Lemma |
|------|------|-------|
| `saw` | 动词 (v) | `see` |
| `saw` | 名词 (n) | `saw`（锯子）|
| `leaves` | 动词 (v) | `leave` |
| `leaves` | 名词 (n) | `leaf` |

**优点**：
- 结果是**合法词汇**，可直接用于词典查询
- 语义保留更完整，错误率低于词干还原

**缺点**：
- 速度较慢（需要查词典和词性标注）
- 需要准确的 POS 标签才能达到最佳效果
- 词汇表外（OOV）的新词无法处理

#### 对比：Stemming vs Lemmatization

| 维度 | 词干还原 (Stemming) | 词形归并 (Lemmatization) |
|------|---------------------|--------------------------|
| **核心思想** | 基于规则截断词缀 | 基于词典还原为规范形式 |
| **结果性质** | 可能是伪词 | 必须是真实单词 |
| **速度** | 极快 | 较慢（需查词典） |
| **准确度** | 低，容易过度截断 | 高，语义保留完整 |
| **POS 依赖** | 不需要 | 需要（或受益于）POS 标签 |
| **代表工具** | PorterStemmer, Snowball | WordNetLemmatizer, spaCy |


#### 使用场景与决策建议

**词干还原更适合**：
- 对速度要求极高的海量文本处理
- 信息检索（IR）中追求**高召回率**的场景
- 早期/传统 NLP 流水线

**词形归并更适合**：
- 需要高语义精度的任务（文本分类、情感分析、机器翻译）
- 下游任务依赖**真实词汇**（如知识图谱、词典匹配）
- 现代 NLP 预处理

**混合策略（工业界常用）**：
> 追求精确率的应用，往往**先基于原始词进行精确匹配**；若失败，再对归一化后的词进行模糊匹配，寻找可能的结果。

- 搜索系统：先用原始词查，再用 lemma/stem 扩展召回
- 这样既能保证精确率，又能提升召回率

## 2.3 情感分析（Sentiment Analysis）

**定义**：识别和提取文本中的主观信息，判断文本表达的情感倾向（正面、负面、中性）。

**应用场景**：
- 用户反馈分析：从评论、社交媒体中收集情感倾向
- 舆情监控：追踪品牌、产品的公众口碑
- 垃圾信息过滤：识别钓鱼邮件或恶意评论的情感操控

**两大类方法对比**：

| 方法 | 核心思想 | 优点 | 缺点 |
|------|---------|------|------|
| **基于规则** | 人工设计启发式规则（如关键词词典） | 无需标注数据，即开即用 | 领域适应性差，规则维护成本高 |
| **基于机器学习** | 从标注数据中学习分类模型 | 泛化能力强，可适应特定领域 | 需要大量标注数据 |

---

### 2.3.1 VADER：基于规则的情感分析器

**原理**：通过预定义的情感词典和语法规则（如标点强化、大小写强化、程度修饰词）来计算文本的综合情感得分。

```python
from vaderSentiment.vaderSentiment import SentimentIntensityAnalyzer
sa = SentimentIntensityAnalyzer()

corpus = [
    "Absolutely perfect! Love it! :-) :-) :-)",
    "Horrible! Completely useless. :(",
    "It was OK. Some good and some bad things."
]
for doc in corpus:
    scores = sa.polarity_scores(doc)
    print('{:+}: {}'.format(scores['compound'], doc))

# +0.9428: Absolutely perfect! Love it! :-) :-) :-)
# -0.8768: Horrible! Completely useless. :(
# -0.1531: It was OK. Some good and some bad things.
```

**输出解读**：
- `compound`：综合情感得分，范围 `[-1, +1]`，越接近 +1 越正面
- `pos` / `neu` / `neg`：正面 / 中性 / 负面情感的比例

**优点**：
- 无需训练，开箱即用
- 对社交媒体文本友好（支持表情符号、缩写、网络用语）
- 速度快，适合实时分析

**⚠️ 风险与局限**：
- 只关注词典中已有的词条，对词典外词汇（OOV）无感知
- 跨语言 / 领域迁移困难，需要为不同语言重建词典
- 无法理解复杂语义（如反讽、隐喻）

**适用场景**：
- 快速原型验证
- 社交媒体实时情感监控
- 对准确度要求不高、但速度要求高的场景

---

### 2.3.2 朴素贝叶斯：基于机器学习的情感分类

**原理**：利用词袋特征和贝叶斯定理，计算文本属于各个情感类别的概率。核心假设是"特征之间相互独立"（词与词之间无关联）。

**数据准备**：

```python
from nlpia.data.loaders import get_data
movies = get_data('hutto_movies')
movies.head().round(2)

# id   sentiment  text
# 1    2.27       The Rock is destined to be the 21st Century's ...
# 2    3.53       The gorgeously elaborate continuation of ''The...
# 3    -0.60      Effective but too tepid biopic
# 4    1.47       If you sometimes like to go to the movies to h...
# 5    1.73       Emerges as something rare, an issue movie that...
```

**文本向量化（词袋模型）**：

```python
import pandas as pd
pd.set_option('display.width', 75)
from nltk.tokenize import casual_tokenize
from collections import Counter

bags_of_words = []
for text in movies.text:
    bags_of_words.append(Counter(casual_tokenize(text)))
df_bows = pd.DataFrame.from_records(bags_of_words).fillna(0).astype(int)
df_bows.shape
# (10605, 20756)
```

- `casual_tokenize`：适合社交媒体文本的分词器，能处理表情符号、缩写
- 最终得到 `(文档数, 词汇表大小)` 的稀疏矩阵

**模型训练与预测**：

```python
from sklearn.naive_bayes import MultinomialNB

nb = MultinomialNB()
nb = nb.fit(df_bows, movies.sentiment > 0)

# 将概率映射回原始情感分数范围（[-4, +4]）
movies['predicted_sentiment'] = nb.predict_proba(df_bows)[:, 1] * 8 - 4
movies['error'] = (movies.predicted_sentiment - movies.sentiment).abs()
movies.error.mean().round(1)

# 二分类标签
movies['sentiment_ispositive'] = (movies.sentiment > 0).astype(int)
movies['predicted_ispositive'] = (movies.predicted_sentiment > 0).astype(int)

# 准确率
(movies.predicted_ispositive == movies.sentiment_ispositive).sum() / len(movies)
# 0.9344648750589345
```

**模型评估**：
- 二分类准确率约 **93.4%**

**优点**：
- 训练和预测速度极快
- 对小规模数据表现良好
- 模型可解释性强（可通过特征概率看出哪些词驱动了分类）

**⚠️ 风险与局限**：
- **特征独立性假设不成立**：`not good` 被当作 `not` 和 `good` 独立处理
- 词袋模型丢失了词序信息
- 对未登录词（OOV）处理能力差
- 需要人工标注数据

---

#### 方法对比：VADER vs 朴素贝叶斯

| 维度 | VADER（规则） | 朴素贝叶斯（机器学习） |
|------|--------------|----------------------|
| **是否需要训练数据** | ❌ 不需要 | ✅ 需要大量标注数据 |
| **速度** | 极快 | 训练较慢，预测快 |
| **领域适应性** | 差（依赖固定词典） | 好（可针对领域重新训练） |
| **对社交媒体的适应性** | 强（内置表情符号规则） | 依赖分词器和特征工程 |
| **可解释性** | 中等（可查词典得分） | 强（特征概率透明） |
| **典型准确率** | 中等 | 高（本例中 93.4%） |

**决策建议**：
- **快速验证 / 原型**：首选 VADER，零成本启动
- **生产环境 / 特定领域**：用朴素贝叶斯或更复杂的模型（如 SVM、BERT）
- **混合策略**：先用机器学习模型打分，对置信度低的样本 fallback 到规则系统

## 关键概念速查

### 2.1 分词基础

| 术语 | 英文 | 含义 |
|------|------|------|
| 分词器 | Tokenizer / Lexer | 将文本切分为词或符号的扫描器 |
| 词条 | Token | 分词后的最小单位 |
| 词汇表 | Vocabulary | 所有不重复词条的集合 |

### 2.2 文本向量化与归一化

| 术语 | 英文 | 含义 |
|------|------|------|
| 独热向量 | One-Hot Vector | 每个词独占一个维度，位置记 1 其余记 0 |
| 词袋模型 | Bag of Words (BOW) | 忽略词序，将文本中各词频累加成的向量 |
| n-gram | n-gram | n 个连续词组成的序列，用于捕捉局部语义 |
| 停用词 | Stop Words | 出现频率极高但语义承载极少的词（如 `the`, `is`） |
| 词干还原 | Stemming | 基于规则截断词缀得到词根（结果可能是伪词） |
| 词形归并 | Lemmatization | 基于词典和词性还原为规范词形（结果是真实单词） |
| 余弦相似度 | Cosine Similarity | 归一化的点积，消除文本长度影响后的相似度度量 |

### 2.3 情感分析

| 术语 | 英文 | 含义 |
|------|------|------|
| 情感分析 | Sentiment Analysis | 识别文本中的主观情感倾向（正面 / 负面 / 中性） |
| VADER | Valence Aware Dictionary and sEntiment Reasoner | 基于规则的情感分析器，对社交媒体文本友好 |
| 朴素贝叶斯 | Naive Bayes | 基于特征独立性假设的概率分类算法 |
