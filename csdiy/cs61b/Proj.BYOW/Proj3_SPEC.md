## 项目简介

BYOW 是一个**2D 基于瓦片的世界探索引擎**设计项目。你将创建一个可生成可探索世界的引擎，用户可以在世界中行走并与物体互动。

> 这是一个**开放式设计项目**，没有"正确答案"，重点在于软件工程实践和代码设计。

### 项目目标
- 学习处理大型代码系统的复杂性
- 经历从构思到展示的完整产品开发周期
- 练习设计文档编写、迭代开发和协作编程

## 项目概述

### 系统架构

你的世界将由 **2D 瓦片网格**组成，使用提供的 `TileEngine` 渲染：

```
world[0][0]          // 左下角瓦片
world[x][y]          // x向右增加，y向上增加
```

所有 `TETile` 必须非空，渲染前填满整个网格。

### 提供的骨架代码

```
proj3/
├── TileEngine/          # 瓦片渲染引擎
│   ├── TERenderer.java  # 渲染方法
│   ├── TETile.java      # 瓦片类型定义
│   └── Tileset.java     # 默认瓦片库
├── Core/                # 核心逻辑（你的代码）
│   ├── AutograderBuddy.java  # getWorldFromInput()
│   ├── Main.java        # 程序入口
│   └── World.java       # 世界逻辑（主要实现处）
└── Utils/               # 工具类
    ├── RandomUtils.java
    └── FileUtils.java
```

### 关键限制
- **不要修改** `TETile.java` 的 `character` 字段和 `character()` 方法
- 添加新的地板/墙壁瓦片时，修改 `isBoundaryTile()` 和 `isGroundTile()`
- **不要使用非 final 的 static 变量**（会导致复杂 bug）
- 只在 `getWorldFromInput()` 中使用标准 Java 库或提供的库
- 不要在 `getWorldFromInput()` 中调用 `System.exit()`

## Phase 1: 3A - 世界生成

### 目标
实现一个随机世界生成器，能够生成有效的、充分随机的世界。

### 世界有效性要求

| 要求 | 说明 |
|------|------|
| 2D网格 | 使用 TileEngine 绘制 |
| 房间和走廊 | 必须包含不同的房间和走廊（可包含户外空间） |
| 矩形房间 | 至少一些房间应该是矩形的（可支持其他形状） |
| 转弯走廊 | 能生成包含转弯的走廊（或等效的相交直走廊），随机世界应有 20%+ 概率生成转弯走廊 |
| 无死胡同 | 不允许死胡同走廊 |
| 墙壁区分 | 墙壁与地板视觉可区分，与未使用空间也可区分 |
| 角落可选 | 角落墙壁是可选的 |
| 连通性 | 房间和走廊应连通，相邻房间/走廊之间无间隙 |
| 可达性 | 所有房间必须可达（无无法进入的房间） |
| 边界检查 | 房间不能超出世界边缘（地板瓦片不能在边缘） |
| 空间利用 | 50%+ 的空间应填充房间和走廊 |

### 随机性要求

- 世界必须**伪随机生成**
- 随机数量的房间和走廊
- 房间和走廊的随机位置
- 房间的随机宽高
- 走廊宽度为 1 瓦片，长度随机
- 每次生成的世界应有显著不同

### 输入处理

程序必须支持 `getWorldFromInput(String input)`：

```
格式: "N#######S"
  N - 新建世界
  # - 种子数字（任意位数）
  S - 种子输入结束

示例: "N3412S" → 使用种子 3412 生成世界
```

#### 主菜单要求

运行 `Main.main()` 时必须显示主菜单，至少有：
- **N** - New world（新建世界）
- **L** - Load world（加载世界）
- **Q** - Quit（退出）

**输入种子流程：**
1. 按 N 进入新建世界
2. 输入数字种子（long 类型，最大 9,223,372,036,854,775,807）
3. UI 显示已输入的数字
4. 按 S 确认，生成世界

#### getWorldFromInput 行为

- 不应渲染任何瓦片（不调用渲染）
- 返回 `TETile[][]` 表示最终世界状态
- 输入字符串不区分大小写（"N" 和 "n" 等价）
- 用于 autograder 测试

### 瓦片渲染

```java
// 创建瓦片
TETile tile = new TETile('·', Color.gray, Color.black, "floor", "floor.png");

// 渲染世界
TERenderer ter = new TERenderer();
ter.initialize(WIDTH, HEIGHT);
ter.renderFrame(world);
```

**图形瓦片：**
- 图像必须是 16x16 像素
- 建议使用 PNG 格式
- 如果图像文件不存在，回退到 Unicode 字符显示
- 每个 `TETile` 必须有唯一字符标识

## Phase 2: 3B - 交互与保存

### 交互要求

1. **角色控制**
   - 使用 **W, A, S, D** 控制角色移动
   - 角色不能穿墙
   - 角色必须能与世界互动（某种方式）

2. **HUD（抬头显示）**
   - 必须显示当前鼠标指向的瓦片描述
   - 不能闪烁
   - 可包含额外信息（如生命值、指令提示等）

3. **保存/加载**
   - 输入 `:Q`（冒号+Q）保存并退出
   - 不区分大小写（`:q` 也可）
   - 重启后按 L 加载，世界状态应完全恢复
   - 包括随机数生成器状态！
   - 只能创建 `.txt` 后缀的文件

4. **确定性**
   - 相同种子 + 相同按键序列 = 完全相同的结果
   - 通过 `getWorldFromInput()` 可回放整个操作序列

### 输入字符串扩展

```
"N543SWWWWAA"      → 新建世界(种子543)，上4次，左2次
"N25SDDWD:Q"       → 新建世界(种子25)，右2次，上，右，保存
"LDDDD"            → 加载，右4次
```

### StdDraw 限制

- 不支持组合键（`:Q` 表示 `:` 然后 `Q`）
- 只能识别产生 char 的按键（不支持方向键、Esc 等）
- 可能不支持长按连续移动
- 不能有依赖实时时间的机制（不利于 autograder）

## Phase 3: 3C - Ambition 进阶功能

实现 **28 分**的进阶功能，其中**必须包含至少一个 21 分的主要功能**。

### 主要功能 (21 分)

| 功能 | 描述 |
|------|------|
| **视野系统** | 只渲染角色视线范围内的瓦片，可用按键开关 |
| **光源系统** | 光源影响渲染，光强随距离递减，光不穿透墙壁，可用按键开关 |
| **追逐实体** | 添加使用课堂搜索算法（如 A*）追逐角色/其他实体的敌人，可显示预测路径 |
| **遭遇系统** | 与实体互动时切换到新界面（如宝可梦式战斗），结束后返回原界面 |
| **视角切换** | 第一人称、等距 2.5D、3D 等视角切换 |
| **战斗系统** | 玩家有生命值，可收集恢复道具，有随机移动敌人造成伤害 |

### 次要功能 (7 分)

| 功能 | 描述 |
|------|------|
| **回放功能** | 重放最近保存的操作序列，视觉上显示所有动作 |
| **多存档槽** | 支持多个存档槽，可用快捷键保存到不同槽位 |
| **不重启新建** | 无需关闭程序即可创建新世界（游戏结束或菜单选项） |
| **自定义角色** | 菜单选项更改角色外观为自定义图片 |
| **主题切换** | 菜单选项或随机确定世界环境/主题 |
| **多语言** | 切换界面语言，默认英语 |
| **鼠标支持** | 主菜单可用鼠标点击 |
| **图像渲染** | 使用图片而非 Unicode 字符渲染 |
| **音乐音效** | 菜单/探索背景音乐和交互音效 |
| **小地图** | 显示整个世界和当前位置的缩略图 |
| **世界旋转** | 90 度旋转世界，相应调整移动键 |
| **鼠标移动** | 点击任意可见方块移动（需实现寻路算法） |
| **双人模式** | 两个玩家同时控制，不同控制方案 |
| **撤销移动** | 撤销最近一次移动（包括加载之前保存的） |

> 可以自由发挥实现未列出的功能！提前在 Ed 上询问确认是否符合标准。

## 需求总结

### 必须实现

- [ ] 主菜单：N（新建）、L（加载）、Q（退出），键盘导航，不区分大小写
- [ ] 新建世界：输入整数种子 + S 键，显示已输入数字，生成并显示世界
- [ ] 伪随机世界：不同种子生成不同世界
- [ ] 3A 视觉要求：房间、走廊、墙壁、连通性、无边缘地板、50%+ 填充
- [ ] WASD 移动角色，不能穿墙
- [ ] `:Q` 保存退出，L 加载恢复完全状态（包括随机数生成器）
- [ ] `getWorldFromInput()` 处理字符串输入，返回 `TETile[][]`，行为与 `main()` 一致
- [ ] 使用提供的 `TileEngine` 和 `StdDraw` 渲染
- [ ] HUD 显示鼠标指向瓦片描述
- [ ] 无实时机制（无输入时无变化）
- [ ] **28 分 Ambition 功能，至少一个主要功能**
